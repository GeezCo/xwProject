1 范围

1.1 标识

　　本文档适用于献微系统的软件需求规格说明。文档标识号：XW-REQ-001，版本号：V1.0.0。本文档针对的计算机软件配置项（CSCI）为献微系统软件，CSCI标识号为XW-CSCI-001。

1.2 系统概述

　　献微系统是一个基于大语言模型（LLM）的军事情报文章撰写与数据分析系统。系统采用前后端分离的三层架构（Vue 3前端 + Spring Boot后端 + Flask算法服务），通过算法服务调用大语言模型，实现智能化的事件抽取和报文融合功能。

　　系统的总体架构如下：

```
┌─────────────────┐     HTTP      ┌─────────────────┐     HTTP      ┌─────────────────┐
│    前端层       │ ───────────▶ │    后端层       │ ───────────▶ │   算法服务层    │
│  Vue 3 + Vite   │              │  Spring Boot    │              │  Flask + LLM    │
│    :8080        │              │    :8081        │              │    :5001        │
└─────────────────┘              └─────────────────┘              └─────────────────┘
                                        │                                │
                                        ▼                                ▼
                                 ┌─────────────┐                ┌─────────────┐
                                 │  MySQL 8.0  │                │  外部服务   │
                                 │   :8512     │                │  LLM/MinIO  │
                                 └─────────────┘                └─────────────┘
```

图1-1 献微系统总体架构图

　　系统包含以下三个核心功能模块：

a) 报文管理模块：负责军事情报报文的查询、筛选、分类管理、详情查看和批量导入，支持文字报和图文报两种模态类型
b) 属性抽取模块：基于大语言模型对报文进行层次化事件抽取，提取时间、地点、主体、动作等关键信息，并通过RexUniNlu服务进行多标签分类
c) 报文融合模块：将多篇相关报文的抽取结果进行融合，通过LLM生成包含标题、摘要、时间线和详细内容的综合分析报告

1.3 文档概述

　　本文档描述了献微系统软件的需求规格，包括CSCI能力需求、外部接口需求、内部接口需求、内部数据需求、计算机资源需求等。本文档的读者对象为软件开发人员、测试人员和项目管理人员。

　　本文档按照GJB 438B-2009《军用软件开发文档通用要求》编写，包含以下主要内容：

a) 需求：详细描述CSCI的各项功能需求、接口需求、数据需求和性能需求
b) 合格性规定：规定验证各项需求的检验方法
c) 需求可追踪性：建立CSCI需求与系统需求之间的追踪关系
