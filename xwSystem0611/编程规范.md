### 3.2 组件规范

- **文件命名**：统一使用 PascalCase（如 `SearchResult.vue`、`FusionResult.vue`）
- **组合式API**：所有新组件必须使用 `<script setup>` 语法
- **TypeScript**：推荐使用，至少为 props 和 emits 提供类型声明

```vue
<script setup>
import { ref, computed, onMounted } from 'vue'
import { useRoute } from 'vue-router'
import { useHomeStore } from '@/stores/modules/home'

// Props定义
const props = defineProps({
  originTextId: { type: Number, required: true }
})

// Emits定义
const emit = defineEmits(['update', 'close'])

// 响应式状态
const loading = ref(false)
const data = ref(null)

// 计算属性
const hasData = computed(() => data.value !== null)

// 生命周期
onMounted(() => {
  fetchData()
})
</script>
```

### 3.3 API调用规范

```javascript
// 使用 utils/request.js 封装的 axios 实例
import request from '@/utils/request'

// GET请求
request.get('/uygur/detail/' + sid)

// POST请求
request.post('/extraction/extract', null, { params: { originTextId, force } })
```

### 3.4 状态管理 (Pinia)

```javascript
// stores/modules/home.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useHomeStore = defineStore('home', () => {
  // State
  const token = ref('')
  const userInfo = ref(null)

  // Getters
  const isLoggedIn = computed(() => !!token.value)

  // Actions
  function setToken(val) {
    token.value = val
  }

  return { token, userInfo, isLoggedIn, setToken }
})
```

- 全局状态使用 Pinia（stores/modules/ 按模块拆分）
- 组件内部状态使用 `ref()` / `reactive()`
- 跨页面传参优先使用路由参数或 Pinia

### 3.5 Element Plus 用法

```javascript
// 按需引入（推荐，减少打包体积）
// vite.config.js 中配置 unplugin-vue-components 自动导入
// 或手动导入：
import { ElMessage, ElMessageBox } from 'element-plus'
```

### 3.6 Vue 3 迁移注意事项

| Vue 2 | Vue 3 | 说明 |
|-------|-------|------|
| `data()` | `ref()` / `reactive()` | 响应式状态 |
| `computed: {}` | `computed()` | 计算属性 |
| `watch: {}` | `watch()` / `watchEffect()` | 侦听器 |
| `methods: {}` | 普通函数 | 方法直接声明 |
| `this.$refs` | `ref()` 模板引用 | DOM引用 |
| `this.$router` | `useRouter()` | 路由实例 |
| `this.$route` | `useRoute()` | 当前路由 |
| `this.$store` | `useXxxStore()` | 状态管理 |
| `created/mounted` | `onMounted()` 等 | 生命周期 |
| `$emit` | `defineEmits()` | 事件发射 |
| `props: {}` | `defineProps()` | 属性定义 |
| Element UI | Element Plus | UI组件库 |
| Vuex | Pinia | 状态管理 |
| Webpack | Vite | 构建工具 |

## 四、后端规范 (Spring Boot)

### 4.1 分层架构

```
Controller → Service(接口) → ServiceImpl(实现) → Mapper → 数据库
                                                    ↓
                                              Mapper XML
```

### 4.2 包结构与命名

| 包名 | 职责 | 命名规则 |
|------|------|---------|
| controller | 接收HTTP请求，参数校验 | `XxxController` |
| service | 业务接口定义 | `XxxService` |
| service.impl | 业务逻辑实现 | `XxxServiceImpl` |
| mapper | 数据访问接口 | `XxxMapper` |
| dto | 数据传输对象 | `XxxDTO` |
| request | 请求参数对象 | `XxxRequest` |
| common | 公共类 | `ResultVO` 等 |
| config | 配置类 | `XxxConfig` |

### 4.3 接口响应格式

```java
// 统一使用 ResultVO 封装响应
ResultVO.success(data)     // code=1, msg="success"
ResultVO.error("错误信息")  // code=0, msg="错误信息"
```

### 4.4 MyBatis规范

- Mapper接口使用 `@Mapper` 注解
- SQL映射文件放在 `resources/mapper/` 目录
- XML文件名与Mapper接口名一致（如 `FusionMapper.xml`）
- DDL脚本放在 `resources/db/` 目录

### 4.5 外部服务调用

- 调用算法服务使用 `HttpURLConnection`
- 超时设置：连接超时10秒，读取超时根据业务（抽取20分钟，融合10分钟）
- 算法服务URL从 `application.yml` 配置读取

## 七、开发流程

### 7.1 需求分析

1. 明确功能需求和优先级
2. 确定涉及的模块（前端/后端/算法/数据库）
3. 如涉及新接口，先定义接口参数和响应格式

### 7.2 数据库变更

1. 编写DDL脚本，放入 `后端/src/main/resources/db/`
2. 先在测试环境执行，验证无误后再应用到生产
3. 新增字段使用 `ALTER TABLE ... ADD COLUMN`，注意设默认值避免影响现有数据

### 7.3 算法开发

1. 修改算法代码（抽取/融合模块）
2. 本地测试：`python app.py` 启动后用 curl 验证接口
3. 确认返回数据结构与后端期望一致

### 7.4 后端开发

1. 开发顺序：DTO → Mapper(接口+XML) → Service(接口+实现) → Controller
2. 调试：`mvn spring-boot:run` 启动，查看日志
3. 用 curl/Postman 测试接口

### 7.5 前端开发

1. 开发顺序：API封装 → 页面组件
2. 调试：`npm run dev` 启动，浏览器测试
3. 确认请求参数和响应处理与后端一致

### 7.6 联调测试

1. 按顺序启动：算法服务(:5001) → 后端(:8081) → 前端(:8080)
2. 完整业务流程测试（从页面操作到数据库落库）
3. 检查边界情况（空数据、超长文本、服务超时等）
