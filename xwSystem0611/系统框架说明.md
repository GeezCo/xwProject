# 献微系统框架说明

## 一、系统概述

献微系统 (Xianwei System) 是一个基于大模型的军事情报文章撰写与数据分析系统，采用前后端分离架构，通过算法服务调用大语言模型实现智能化的事件抽取和报文融合功能。

### 系统架构图

```
┌─────────────────┐     HTTP      ┌─────────────────┐     HTTP      ┌─────────────────┐
│   前端层        │ ───────────▶ │   后端层        │ ───────────▶ │  算法服务层     │
│  Vue 3 + Vite   │              │  Spring Boot    │              │   Flask + LLM   │
│    :8080        │              │     :8081       │              │     :5001       │
└─────────────────┘              └─────────────────┘              └─────────────────┘
                                        │                                  │
                                        ▼                                  ▼
                                 ┌─────────────┐                  ┌─────────────┐
                                 │  MySQL 8.0  │                  │  外部服务   │
                                 │   :8512     │                  │  LLM/MinIO  │
                                 └─────────────┘                  └─────────────┘
```

### 2.3 关键业务流程

#### 报文管理流程
```
用户访问 searchResult.vue
  ↓
加载分类树 + 报文列表
  ↓
用户操作：
  ├─ 点击分类 → 筛选报文
  ├─ 搜索关键词 → 模糊查询
  ├─ 点击报文 → 右侧展示详情
  │   ├─ 文字报：纯文本展示
  │   └─ 图文报：文本 + 图片网格（MinIO加载）
  └─ 点击"属性抽取" → 调用算法服务（2-5分钟）
```

#### 报文融合流程
```
选择多篇报文 → 跳转 fusionResult.vue
  ↓
左侧展示源报文列表（含抽取结果）
  ↓
点击"生成融合报告" → 调用算法服务
  ↓
右侧展示融合报告：标题/摘要/时间线/详细内容/实体列表
  ↓
保存报告 → 存入数据库 / 导出 → PDF/Word格式
```

#### 每日要情流程
```
用户访问 DailyBrief.vue → 选择日期和主题
  ↓
查询已分析的事件（后台定时任务已完成分析）
  ↓
按关键词筛选 → 五大主题 Tab 展示
  ↓
可选操作：手动触发分析任务（异步执行）
```

### 3.3 定时任务机制 ✨ 新增

**DailyAnalysisTask**：
- **执行时间**：每日凌晨 2:00（Cron: `0 0 2 * * ?`）
- **执行逻辑**：
  1. 查询前一天新增报文
  2. 逐条调用算法服务 `/eventSplit`
  3. 解析事件数据并保存到 `event_analysis` 表
  4. 已分析报文自动跳过（基于联合唯一索引）

### 4.3 核心算法流程

#### 属性抽取算法（V3版本）
```
输入文本 → 预处理 → 构建Prompt → 调用LLM → 三重JSON解析 → 后处理 → 输出事件列表
```

**特性**：
- 输入预处理（转义引号，防止JSON解析失败）
- 一次性抽取+分类（减少LLM调用次数）
- 三重JSON解析策略（直接解析 → Markdown代码块 → 花括号计数）
- 状态机修复（处理不完整JSON）

#### 报文融合算法
```
Step 1: 数据预处理 → 统一字段格式
Step 2: 信息整合（代码）→ 合并时间线/实体/标签
Step 3: LLM生成 → 标题/摘要/详细内容
Step 4: 结果组装 → 构建目标列表表格 + 输出融合报告
```

#### 事件拆分算法 ✨ 新增
```
输入文本 → 识别独立事件 → 逐事件深度分析（时间/空间/主体/行为/影响/关联） → 输出事件列表
```

### 4.4 LLM配置

```
服务商: Paratera（并行科技）
地址: https://llmapi.paratera.com
主模型: Qwen3.5-122B-A10B
  - 参数量: 122B
  - 上下文: 32K tokens
  - 用途: 属性抽取、报文融合、事件拆分
  - 温度: 0.1-0.3（低温度保证稳定性）

兜底模型: GLM-5
  - 用途: Qwen不可用时自动切换
```

## 五、数据层架构

### 5.1 核心数据表

#### origin_text（原始报文表）
```sql
字段: sid, title, content, times, type, modal_type, images, is_extracted
索引: idx_type, idx_modal_type, idx_is_extracted
```

#### text_type（分类目录表）
```sql
字段: id, type_name, parent_id
索引: idx_parent_id
结构: 树形结构（一级分类 + 二级分类）
```

#### extraction_result（抽取结果表）
```sql
字段: id, origin_text_id, events_json, labels_json, extraction_time
唯一索引: uk_origin_text_id
```

#### fusion_report（融合报告表）
```sql
字段: id, fusion_id, title, summary, timeline_json, content, entities_json, labels_json, source_ids
索引: idx_fusion_id
```

#### event_analysis（事件分析表）✨ 新增
```sql
字段: id, origin_text_id, event_time, event_location, event_content, event_analysis, analysis_date
索引: idx_origin_text_id, idx_analysis_date
唯一约束: uk_origin_event (origin_text_id, event_content)
```

## 七、关键技术模式

### 7.1 多编码文件读取
依次尝试 utf-8 → gbk → gb2312 → utf-16

### 7.2 JSON响应解析（三重策略）
1. 直接解析
2. 提取markdown代码块
3. 花括号计数提取完整对象

### 7.3 API调用重试机制
最多3次重试，间隔2秒

### 7.4 长超时设置
- 前端: 20分钟（1200000ms）
- 后端: 20分钟（适配LLM处理）

### 7.5 缓存策略
- 前端路由缓存（searchResult页面）
- 后端抽取结果缓存（is_extracted标记）
- 数据库查询缓存（event_analysis表）

### 8.2 数据库索引优化
- origin_text: idx_type, idx_modal_type, idx_is_extracted
- extraction_result: uk_origin_text_id
- event_analysis: idx_origin_text_id, idx_analysis_date

### 8.3 连接池配置
- 初始连接数: 5
- 最小空闲连接: 5
- 最大活跃连接: 20

## 十、系统特点

### 10.1 核心优势
- **三层架构清晰**: 前端(Vue 3) → 后端(Spring Boot) → 算法(Flask) → LLM
- **模块化设计**: 6大核心模块，职责明确，低耦合
- **智能化核心**: 基于大模型的事件抽取和报文融合
- **高可用性**: 重试机制、缓存策略、长超时设置
- **后台任务**: 定时分析解耦前后端，提升用户体验

### 10.2 技术亮点
- **V3抽取器**: 状态机修复 + 输入预处理 + 一次性抽取分类
- **融合算法**: 代码整合 + LLM生成，兼顾效率与质量
- **动态配置**: MinIO地址前端动态获取，迁移无需重新构建
- **图文报支持**: 图片存储 + 网格展示 + 懒加载
- **每日要情**: 后台定时分析 + 前端秒级查询 + 关键词智能筛选
