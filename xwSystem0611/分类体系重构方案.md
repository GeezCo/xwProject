# 献微系统 - 分类体系重构方案

## 一、重构目标

将 `text_type` 表从二级固定分类重构为**多级树形分类**，支持根据报文的 `sendUnitName` 字段自动创建和分配分类节点。

---

## 二、数据库重构

### 2.1 表结构变更

**原表结构 (旧)**：
```sql
CREATE TABLE text_type (
  sid INT PRIMARY KEY AUTO_INCREMENT,
  id INT,
  type_name VARCHAR(100),
  parent_id INT
);
```

**新表结构**：
```sql
CREATE TABLE text_type (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  parent_id BIGINT DEFAULT NULL,
  level TINYINT NOT NULL DEFAULT 1,
  full_path VARCHAR(500) NOT NULL,
  sort_order INT DEFAULT 0,
  is_leaf TINYINT(1) DEFAULT 0,
  description VARCHAR(255),
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  
  INDEX idx_parent_id (parent_id),
  INDEX idx_level (level),
  INDEX idx_full_path (full_path),
  INDEX idx_is_leaf (is_leaf),
  UNIQUE KEY uk_name_parent (name, parent_id)
);
```

**字段说明**：
- `id`: 主键，BIGINT 自增
- `name`: 分类名称（原 `type_name`）
- `parent_id`: 父分类 ID
- `level`: 层级深度（1=根分类）
- `full_path`: 完整路径（如 "根分类/美国/国防部"）
- `sort_order`: 排序权重
- `is_leaf`: 是否叶子节点（1=叶子，0=分支）
- `description`: 分类描述
- `create_time`, `update_time`: 时间戳

### 2.2 数据迁移脚本

**位置**: `后端/src/main/resources/db/migration/restructure_text_type.sql`

**执行步骤**：
```sql
-- 1. 备份旧表
CREATE TABLE text_type_backup AS SELECT * FROM text_type;

-- 2. 清空旧表
TRUNCATE TABLE text_type;

-- 3. 删除旧表
DROP TABLE text_type;

-- 4. 创建新表（见上方新表结构）

-- 5. 初始化根节点
INSERT INTO text_type (name, parent_id, level, full_path, sort_order, is_leaf, description)
VALUES 
  ('根分类', NULL, 1, '根分类', 0, 0, '系统根节点'),
  ('未分类', 1, 2, '根分类/未分类', 999, 1, '新导入无匹配单位的默认归属');

-- 6. 迁移旧数据（如果需要）
-- 旧表没有层级结构，全部数据归到"未分类"下
UPDATE origin_text SET type = 2 WHERE type IN (SELECT DISTINCT id FROM text_type_backup);
```

### 3.4 修改导入逻辑

#### UygurServiceImpl.java 的 importFromJsonl 方法

**修改前**：
```java
// 查询固定的二级分类
TextTypeDTO category = uygurMapper.getCategoryByNameAndParent(categoryName, parentId);
Integer categoryId = category.getId();
```

**修改后**：
```java
// 遍历每条报文
for (OriginTextDTO text : texts) {
    // 1. 从报文中提取 sendUnitName
    String sendUnitName = extractSendUnitName(text.getContent());
    
    // 2. 自动创建或获取分类节点
    TextTypeDTO category = categoryService.getOrCreateCategoryByUnitName(sendUnitName);
    
    // 3. 设置报文的分类
    text.setType(category.getId());
    
    // 4. 保存报文
    uygurMapper.insertOriginText(text);
}
```

**extractSendUnitName 实现**：
```java
private String extractSendUnitName(String content) {
    // 正则提取 "【送报单位】：XXX" 或 "送报单位: XXX"
    Pattern pattern = Pattern.compile("[【\\[]?送报单位[】\\]]?[:\\s：]+([^\\n]+)");
    Matcher matcher = pattern.matcher(content);
    if (matcher.find()) {
        return matcher.group(1).trim();
    }
    return null; // 未找到则返回 null，归入"未分类"
}
```

## 八、注意事项

1. **字段名变更**：
   - 所有 `sid` → `id`
   - 所有 `type_name` → `name`

2. **编码问题**：
   - application.yml 已配置 UTF-8
   - multipart 表单参数需正确处理中文

3. **性能优化**：
   - `full_path` 字段建立索引
   - 分类树查询使用递归 CTE（MySQL 8.0+）

4. **数据一致性**：
   - 删除分类时必须处理关联报文
   - 移动分类时需重新计算 `full_path`
