### 模块4：情报分析（线框图 07/08/09）

> 新增 Controller: `IntelligenceController`，前缀 `/api/intelligence`
> 部分接口需要**算法服务**支持（实体关系构建、事件因果推理）。

#### 4.1 情报关系图谱（线框图07）

| # | 方法 | 路径 | 功能 | 参数 | 需算法 |
|---|------|------|------|------|--------|
| I1 | GET | /api/intelligence/graph | 获取知识图谱数据 | entityTypes, relationTypes, timeRange | **是** |
| I2 | GET | /api/intelligence/entity/{id} | 实体详情+关联 | 无 | 否 |
| I3 | POST | /api/intelligence/graph/build | 触发图谱构建/更新 | sourceIds(报文范围) | **是** |

**算法服务新增端点**：

| # | 方法 | 路径 | 功能 | 说明 |
|---|------|------|------|------|
| A1 | POST | /intelligence/build_graph | 构建实体关系图谱 | 输入：报文列表+抽取结果 → 输出：nodes+edges |

**说明**：
- 图谱数据来源于已有的 `extraction_result` 表（events_json 中的 subject/location/action）
- 算法服务负责：实体消歧（"拜登"="美国总统拜登"）、关系推断（从事件中推断隶属/指挥等关系）、图谱剪枝
- 后端缓存构建好的图谱数据，前端查询时直接返回

#### 4.2 情报时间轴（线框图08）

| # | 方法 | 路径 | 功能 | 参数 | 需算法 |
|---|------|------|------|------|--------|
| I4 | GET | /api/intelligence/timeline | 获取时间轴事件列表 | eventTypes, regions, importance, sources, timeRange | 部分 |
| I5 | GET | /api/intelligence/event/{id} | 事件详情 | 无 | 否 |
| I6 | POST | /api/intelligence/timeline/build | 构建时间轴（因果关系推断） | sourceIds | **是** |

**算法服务新增端点**：

| # | 方法 | 路径 | 功能 | 说明 |
|---|------|------|------|------|
| A2 | POST | /intelligence/build_timeline | 构建事件时间轴+因果关系 | 输入：事件列表 → 输出：events+relations(因果链) |

**说明**：
- 事件数据同样来源于 extraction_result（time/location/action）
- 算法服务负责：事件重要性评估（LLM打分）、因果关系推断（LLM判断"导致/影响/相关"）、地区归类
- 时间轴和图谱共享底层事件数据，需要统一的 `intelligence_event` 中间表

#### 4.3 地理热力图（线框图09）

| # | 方法 | 路径 | 功能 | 参数 | 需算法 |
|---|------|------|------|------|--------|
| I7 | GET | /api/intelligence/heatmap | 获取热力图数据 | eventTypes, heatMetric, timeRange | 否 |
| I8 | GET | /api/intelligence/region | 区域详情 | regionName | 否 |

**说明**：
- 热力值计算逻辑（事件数量/重要性/活跃度）可在后端 Java 层完成
- 需要事件的经纬度坐标 → 需要**地理编码服务**（地名→坐标），可调用外部 API 或本地数据库
- 建议在 `intelligence_event` 表中预存 lat/lng 字段

## 三、需新增数据库表

### 3.1 情报分析模块

```sql
-- 情报实体表（图谱节点）
CREATE TABLE intelligence_entity (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    entity_id   VARCHAR(64) UNIQUE NOT NULL COMMENT '实体唯一标识 如 person_001',
    label       VARCHAR(200) NOT NULL COMMENT '实体名称',
    type        VARCHAR(50) NOT NULL COMMENT '类型: 人物/组织/部队/军舰/战机/武器/地点/事件',
    count       INT DEFAULT 0 COMMENT '出现次数',
    first_seen  DATETIME COMMENT '首次出现时间',
    last_seen   DATETIME COMMENT '最近出现时间',
    properties  JSON COMMENT '扩展属性 如 {"职位":"总统","国籍":"美国"}',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 实体关系表（图谱边）
CREATE TABLE intelligence_relation (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    source_id   VARCHAR(64) NOT NULL COMMENT '源实体ID',
    target_id   VARCHAR(64) NOT NULL COMMENT '目标实体ID',
    type        VARCHAR(50) NOT NULL COMMENT '关系类型: 隶属/参与/位于/使用/指挥/攻击',
    weight      INT DEFAULT 1 COMMENT '关系权重',
    properties  JSON COMMENT '关系属性',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_source (source_id),
    INDEX idx_target (target_id)
);

-- 情报事件表（时间轴/热力图共用）
CREATE TABLE intelligence_event (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    event_id    VARCHAR(64) UNIQUE NOT NULL,
    title       VARCHAR(500) NOT NULL,
    time        DATETIME NOT NULL,
    type        VARCHAR(50) COMMENT '军事/政治/经济/外交/恐怖/冲突',
    importance  VARCHAR(10) COMMENT '高/中/低',
    location    JSON COMMENT '地点名称数组',
    lat         DOUBLE COMMENT '纬度（用于热力图）',
    lng         DOUBLE COMMENT '经度（用于热力图）',
    region      VARCHAR(50) COMMENT '所属地区: 东亚/中东/欧洲/美洲/非洲',
    summary     TEXT COMMENT 'AI生成摘要',
    entities    JSON COMMENT '关键实体',
    source      VARCHAR(50) COMMENT '数据来源: 新闻/社交/官方',
    source_report_ids JSON COMMENT '关联的原始报文SID数组',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_time (time),
    INDEX idx_type (type),
    INDEX idx_region (region)
);

-- 事件因果关系表
CREATE TABLE intelligence_event_relation (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    from_event  VARCHAR(64) NOT NULL,
```

### 算法技术栈说明

| 能力 | 推荐技术 | 说明 |
|------|----------|------|
| 实体消歧 | LLM (GLM-5/Qwen3.5) | 基于上下文判断两个实体名是否指同一对象 |
| 关系推断 | LLM | 从事件描述中推断实体间关系类型 |
| 因果推断 | LLM | 判断事件之间的因果/时序关系 |
| 轨迹预测 | 卡尔曼滤波 / LSTM | 基于历史轨迹点预测未来路线 |
| 聚类分析 | DBSCAN / K-Means (scikit-learn) | 识别高频驻留区域和常用路线 |
| 异常检测 | Isolation Forest / LOF (scikit-learn) | 检测偏离常规模式的行为 |
| 相似度匹配 | 特征向量 + 余弦相似度 | 基于行为特征匹配相似目标 |
| 方案生成 | LLM | 基于模板和约束条件生成侦察方案 |
| 方案评估 | LLM + 规则引擎 | 多维度打分（可行性/风险/效率/时效） |

## 七、实施优先级建议

### 第一优先级：数据看板（工作量小，价值高）

- **8个统计接口**，全部基于现有4张表聚合查询
- 无需新建表、无需算法服务、无需 WebSocket
- 预估工作量：**1-2天**（后端） + **2-3天**（前端 ECharts 集成）
- 依赖：无新增依赖

### 第二优先级：情报分析三视图（数据基础已有）

- 数据来源于已有的 `extraction_result` 表，需要构建中间表 + 算法处理
- 图谱/时间轴/热力图三个前端页面共享底层事件数据
- 预估工作量：
  - 数据库 + 后端接口：**3-5天**
  - 算法服务（图谱构建+因果推理）：**5-7天**
  - 前端（Cytoscape + Vis.js + Leaflet）：**7-10天**
- 依赖：地理编码服务（地名→经纬度）

### 第三优先级：轨迹分析（全新数据域）

- 需要全新的数据模型、数据采集管道、WebSocket 实时推送
- 算法复杂度高（预测、聚类、异常检测）
- 预估工作量：
  - 数据库 + 后端接口 + WebSocket：**5-7天**
  - 算法服务（6个端点）：**7-10天**
  - 前端（4个页面 + 地图）：**10-15天**
- 依赖：轨迹数据源（模拟数据 or 外部对接）

### 第四优先级：侦察筹划（业务最复杂）

- 涉及完整的审批工作流、资源调度、AI 辅助决策
- 需要与轨迹分析模块联动（态势数据共享）
- 预估工作量：
  - 数据库 + 后端接口 + WebSocket：**5-7天**
  - 算法服务（4个LLM端点）：**5-7天**
  - 前端（4个页面 + 向导 + 地图）：**10-15天**
- 依赖：轨迹分析模块提供态势数据
