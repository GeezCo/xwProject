一、概述
================================================================================

1.1 项目背景
------------
当前文本事件抽取面临两个挑战：
- 纯规则方法：各要素独立抽取，无法建立事件内要素的关联关系
- 纯LLM方法：每段落都调用大模型，成本高、速度慢

1.2 设计目标
------------
设计并实现层次化事件抽取算法，核心目标：
(1) 抽取原子化事件：每个事件包含关联的四要素（时间、地点、主体、行为）
(2) 降低LLM调用成本：通过规则筛选，减少不必要的API调用
(3) 保证抽取质量：不遗漏事件，不重复抽取

1.3 核心创新
------------
采用两层架构：
- 第一层：轻量级规则筛选（快速、低成本）
- 第二层：LLM精确抽取（只处理有价值的段落）

三、详细设计
================================================================================

3.1 第一层：规则筛选器
----------------------

3.1.1 筛选策略
--------------
采用"中等平衡"策略，兼顾召回率和成本：

    判断条件：有行为动词 AND (有时间表达 OR 有主体)

    - 必须条件：行为动词（表示、指出、会晤、签署等）
    - 辅助条件：时间表达 或 主体（人物/组织）

3.1.2 特征检测规则
------------------

四、接口设计
================================================================================

4.1 命令行接口
--------------
用法：
    python hierarchical_extractor.py <input.txt> [output.json] [api_key]

参数：
    input.txt   : 必需，输入文本文件路径
    output.json : 可选，输出JSON文件路径
    api_key     : 可选，API密钥

示例：
    python hierarchical_extractor.py sample.txt
    python hierarchical_extractor.py input.txt output.json sk-xxx

4.2 编程接口
------------
from hierarchical_extractor import HierarchicalEventExtractor

# 初始化
extractor = HierarchicalEventExtractor(api_key="sk-xxx")

# 执行抽取
result = extractor.extract_all(text)

# 访问结果
events = result['events']              # 事件列表
skipped = result['skipped_paragraphs'] # 跳过的段落
