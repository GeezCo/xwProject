# 数据抽取-新

**当前生产版**事件抽取算法，使用 **V3 版本**（输入预处理 + 三重JSON解析 + 状态机修复）。

## 核心文件

| 文件 | 说明 |
|------|------|
| **llm_event_extractor_v3.py** | **V3事件抽取器（当前生产版本）**：输入预处理防御引号问题，三重JSON解析，状态机修复 |
| llm_event_extractor.py | LLM 事件抽取底层，含动态提示词生成、JSON 解析、重试机制 |
| hierarchical_extractor.py | 层次化事件抽取（可选）：规则筛选 → LLM 精确抽取，节省约 50% API 调用 |
| classify_client.py | RexUniNlu 分类客户端（已弃用，保留兼容） |
| config.json | **核心配置文件**：LLM 连接（base_url/model/api_key）、实体分类标签列表 |
| config.example.json | 配置文件模板 |
| element_extractor.py | 纯规则抽取（正则 + 分词），无需 LLM，速度快但精度有限 |
| requirements.txt | Python 依赖清单 |

## V3版本特性

1. **输入预处理**：自动清理引号，防止JSON解析失败
2. **三重JSON解析**：直接解析 → Markdown代码块提取 → 花括号计数
3. **状态机修复**：处理不完整的JSON响应
4. **继承架构**：继承自 `LLMEventExtractor`，仅覆盖文本预处理逻辑

## 算法服务集成

`算法/算法服务/app.py` 使用 V3 版本：

```python
from llm_event_extractor_v3 import LLMEventExtractorV3

extractor = LLMEventExtractorV3()
```

所有通过 Flask API (`POST /extract`) 的抽取请求均使用 V3 版本。
