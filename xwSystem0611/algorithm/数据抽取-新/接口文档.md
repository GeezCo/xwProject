# 数据抽取-新 接口文档

## 概述

新版事件抽取模块，相比旧版的主要变化：
1. **主体分类细化**：旧版单一 `subject` 字段拆分为 11 个可配置的分类字段（船只、飞机、无人机、导弹等）
2. **分类可配置**：主体分类在 `config.json` 的 `subject_categories` 中定义，增删分类无需改代码
3. **移除标签分类**：不再调用 RexUniNlu 分类服务，由 LLM 在抽取时直接归类
4. **V3版本优化**：输入预处理防御引号问题，三重JSON解析策略，状态机修复

**当前生产版本**：`llm_event_extractor_v3.py` (LLMEventExtractorV3)  
**当前使用模型**：Qwen/Qwen3.5-35B-A3B-w8a8-mtp（通义千问3.5，35B参数）

---

## 一、处理架构

### V3版本（当前生产）

```
输入文本 → 段落分割
  → 输入预处理（V3特性）
     - 清理引号：\" → "，中文引号 → ASCII引号
     - 移除配对引号内容，保留文本
     - 移除残留引号
  → LLM 精确抽取 (llm_event_extractor.py)
     - 动态提示词（根据 config.json 中的 subject_categories 生成）
     - 调用大模型抽取结构化事件
     - 三重 JSON 解析容错（直接解析 → Markdown代码块 → 花括号计数）
  → 合并去重 → 输出 JSON
```

### 层次化版本（可选）

```
输入文本 → 段落分割
  → 第一层：规则筛选 (hierarchical_extractor.py)
     - 正则匹配时间表达式
     - jieba 分词检测主体/行为动词
     - 有主体 或 有时间 → 进入第二层，否则跳过
  → 第二层：LLM 精确抽取 (llm_event_extractor.py)
     - 动态提示词生成
     - 调用大模型抽取结构化事件
     - 三重 JSON 解析容错
  → 合并去重 → 输出 JSON
```

## 二、Python 调用接口

### 2.1 V3抽取器（当前生产版本，推荐）

```python
from llm_event_extractor_v3 import LLMEventExtractorV3

extractor = LLMEventExtractorV3()
result = extractor.extract_all("报文文本内容...")
```

**特点**：
- 输入预处理：自动清理引号，防止JSON解析失败
- 三重JSON解析：直接解析 → Markdown代码块提取 → 花括号计数
- 状态机修复：处理不完整的JSON响应
- 继承自 `LLMEventExtractor`，仅覆盖文本预处理逻辑

### 2.2 层次化抽取（规则筛选+LLM）

```python
from hierarchical_extractor import HierarchicalEventExtractor

extractor = HierarchicalEventExtractor()
result = extractor.extract_all("报文文本内容...")
```

**特点**：节省约50% LLM调用成本，适合事件稀疏的文本

### 2.3 纯 LLM 抽取（无规则筛选）

```python
from llm_event_extractor import LLMEventExtractor

extractor = LLMEventExtractor()
result = extractor.extract_all("报文文本内容...")
```

**入参**：
| 参数 | 类型 | 说明 |
|------|------|------|
| text | str | 待抽取的报文文本 |

**返回值** `result`：
```json
{
  "extraction_time": "2026-04-30 21:00:00",
  "text_length": 416,
  "paragraph_count": 1,
  "llm_calls": 1,
  "llm_calls_saved": 0,
  "total_events": 9,
  "subject_categories": ["船只", "飞机", "无人机", "导弹", "雷达", "火炮", "坦克/装甲车辆", "军事基地", "部队番号", "人物", "组织"],
  "events": [...],
  "skipped_paragraphs": [...]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| extraction_time | str | 抽取时间 |
| text_length | int | 原文长度（字符） |
| paragraph_count | int | 段落总数 |
| llm_calls | int | 实际调用 LLM 次数 |
| llm_calls_saved | int | 第一层筛选跳过的段落数（节省的 LLM 调用，仅层次化版本） |
| total_events | int | 抽取到的事件总数 |
| subject_categories | list | 当前使用的主体分类名称列表 |
| events | list | 事件列表，详见下方 |
| skipped_paragraphs | list | 被跳过的段落列表（含跳过原因，仅层次化版本） |

### 2.4 纯 LLM 抽取（无规则筛选）

```python
from llm_event_extractor import LLMEventExtractor

extractor = LLMEventExtractor()
result = extractor.extract_all("报文文本内容...")
```

返回值结构同上，但无 `llm_calls_saved` 和 `skipped_paragraphs` 字段。

## 三、事件数据结构

每个事件对象的字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| event_id | int | 是 | 事件编号（从1开始） |
| time | str | 是 | 事件时间，缺失时为 `"未抽取"` |
| location | list[str] | 是 | 纯地理位置列表（城市、国家、海域等） |
| action | str | 是 | 核心行为描述，缺失时为 `"未抽取"` |
| original_text | str | 是 | 原文句子 |
| paragraph_id | int | 是 | 所属段落编号（从0开始） |
| 船只 | list[str] | 否 | 舰船类实体 |
| 飞机 | list[str] | 否 | 有人驾驶飞机实体 |
| 无人机 | list[str] | 否 | 无人飞行器实体 |
| 导弹 | list[str] | 否 | 导弹及反导系统实体 |
| 雷达 | list[str] | 否 | 雷达及电子设备实体 |
| 火炮 | list[str] | 否 | 火炮及火箭炮系统实体 |
| 坦克/装甲车辆 | list[str] | 否 | 坦克及装甲车辆实体 |
| 军事基地 | list[str] | 否 | 军事设施实体 |
| 部队番号 | list[str] | 否 | 部队编制单位实体 |
| 人物 | list[str] | 否 | 人物实体 |
| 组织 | list[str] | 否 | 组织机构实体 |

> 主体分类字段为动态字段，由 `config.json` 中的 `subject_categories` 决定。LLM 未识别到的分类字段不会出现在事件中。

**示例**：
```json
{
  "event_id": 1,
  "time": "2026年1月21日",
  "location": ["西太平洋"],
  "军事基地": ["横须贺海军基地"],
  "飞机": ["F-15 战斗机"],
  "action": "从横须贺海军基地起飞赴西太平洋空域活动",
  "original_text": "F-15 战斗机从横须贺海军基地起飞赴西太平洋空域活动；",
  "paragraph_id": 0
}
```

## 五、配置文件 (config.json)

### 5.1 LLM 配置

**当前生产配置**：
```json
{
  "llm": {
    "base_url": "http://36.141.21.176:8513/v1",
    "model": "Qwen/Qwen3.5-35B-A3B-w8a8-mtp",
    "api_key": "sk-xxx",
    "temperature": 0.1,
    "max_tokens": 16000
  }
}
```

| 字段 | 说明 | 当前值 |
|------|------|--------|
| base_url | LLM API 地址（OpenAI 兼容格式） | http://36.141.21.176:8513/v1 |
| model | 模型名称 | Qwen/Qwen3.5-35B-A3B-w8a8-mtp |
| api_key | API 密钥 | sk-Sf3cCx7aSWyk_4KUFoi8Tw |
| temperature | 温度参数（越低越稳定，推荐 0.1） | 0.1 |
| max_tokens | 最大输出 token 数 | 16000 |

### 5.2 抽取配置

```json
{
  "extraction": {
    "min_paragraph_length": 20,
    "retry_count": 3,
    "retry_delay": 2
  }
}
```

| 字段 | 说明 |
|------|------|
| min_paragraph_length | 段落最小字符数，低于此长度的段落被过滤 |
| retry_count | LLM 调用失败重试次数 |
| retry_delay | 重试间隔（秒） |

### 5.3 主体分类配置 (subject_categories)

```json
{
  "subject_categories": {
    "分类名": {
      "description": "分类描述，告诉 LLM 这个分类的边界和范围",
      "examples": ["实体样例1", "实体样例2"]
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| 键名 | 分类名称，直接作为事件 JSON 中的字段名 |
| description | 分类描述，写入 LLM 提示词，定义分类边界 |
| examples | 具体实体样例，写入 LLM 提示词，作为 few-shot 示例 |

**增删分类**：直接在 `subject_categories` 中增删条目即可，代码自动适配，无需修改任何 Python 文件。

当前 11 个分类：船只、飞机、无人机、导弹、雷达、火炮、坦克/装甲车辆、军事基地、部队番号、人物、组织

## 八、版本说明

### V3版本特性（当前生产）

1. **输入预处理**：
   - 清理转义引号：`\"` → `"`
   - 统一中文引号：`"` `"` → `"`
   - 移除配对引号：`"xxx"` → `xxx`
   - 移除残留引号

2. **三重JSON解析策略**：
   - 策略1：直接 `json.loads()`
   - 策略2：提取 Markdown 代码块 ` ```json ... ``` `
   - 策略3：花括号计数提取完整JSON对象

3. **状态机修复**：处理不完整的JSON响应

4. **继承关系**：`LLMEventExtractorV3` 继承自 `LLMEventExtractor`，仅覆盖 `extract_events_from_paragraph()` 方法

### 算法服务集成

`算法/算法服务/app.py` 第 33 行：
```python
from llm_event_extractor_v3 import LLMEventExtractorV3
```

所有通过 Flask API (`POST /extract`) 的抽取请求均使用 V3 版本。
