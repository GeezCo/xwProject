# 事件拆分接口文档

## 接口概述

**接口名称**：事件拆分与分析（Event Split）

**接口路径**：`POST /eventSplit`

**服务地址**：`http://localhost:5001/eventSplit`

**功能描述**：
将包含多个事件的报文文本进行智能拆分，提取出每个独立事件的时间、地点、内容等信息，并对每个事件进行深度分析，同时生成整体综合分析。

**应用场景**：
- 复杂情报报文的结构化处理
- 多事件新闻稿的自动拆分
- 事件时间线构建
- 情报分析与决策支持

---

## 请求参数

### HTTP Headers

```
Content-Type: application/json
```

### Request Body

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| text | string | 是 | 待拆分的报文原文，支持多事件混合文本 |


### 响应字段说明

#### 顶层字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| events | array | 事件列表，包含所有拆分出的独立事件 |
| overall_analysis | string | 整体综合分析（纯文本，包含趋势判断、风险预警、战略建议等） |

#### events[] - 事件列表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| time | string | 事件发生时间 |
| location | string | 事件发生地点 |
| event_content | string | 事件核心内容（一句话概括） |
| event_analysis | string | 单事件深度分析（纯文本，包含时间、空间、主体、行为、影响、关联等多维度分析） |

## 调用示例

### Python 示例

```python
import requests
import json

def event_split(text):
    url = "http://localhost:5001/eventSplit"
    headers = {"Content-Type": "application/json"}
    data = {"text": text}
    
    response = requests.post(url, headers=headers, json=data)
    result = response.json()
    
    if 'events' in result:
        print(f"拆分成功，共识别 {len(result['events'])} 个事件")
        for i, event in enumerate(result['events'], 1):
            print(f"\n=== 事件 {i} ===")
            print(f"时间: {event['time']}")
            print(f"地点: {event['location']}")
            print(f"内容: {event['event_content']}")
            print(f"分析: {event['event_analysis'][:100]}...")  # 仅显示前100字
        
        print(f"\n=== 整体分析 ===")
        print(result['overall_analysis'][:200] + "...")  # 仅显示前200字
    else:
        print(f"拆分失败: {result.get('msg', '未知错误')}")
    
    return result

# 使用示例
text = """
2024年3月15日，美国总统拜登在白宫签署了一项针对人工智能的行政命令。
同日下午，欧盟委员会在布鲁塞尔宣布通过《人工智能法案》最终版本。
"""

result = event_split(text)
```

### JavaScript 示例

```javascript
async function eventSplit(text) {
  const url = 'http://localhost:5001/eventSplit';
  
  try {
    const response = await fetch(url, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ text })
    });
    
    const result = await response.json();
    
    if (result.events) {
      console.log(`拆分成功，共识别 ${result.events.length} 个事件`);

        console.log(`\n=== 事件 ${index + 1} ===`);

        console.log(`地点: ${event.location}`);
        console.log(`内容: ${event.event_content}`);

    return result;

  }
}
```

## 分析维度说明

### event_analysis - 单事件分析

每个事件的 `event_analysis` 字段包含多维度深度分析（纯文本形式），通常涵盖：

- **时间维度**：时间节点的重要性、历史背景、时序关系
- **空间维度**：地理位置的战略意义、区域影响
- **主体维度**：参与实体（人物、组织）及其角色、动机
- **行为维度**：动作类型、实施方式、目的意图
- **影响维度**：直接影响、潜在后果、受影响方
- **关联维度**：与其他事件的关系、因果链、发展趋势

### overall_analysis - 整体综合分析

`overall_analysis` 字段提供跨事件的宏观视角（纯文本形式），通常包含：

- **综合摘要**：所有事件的总体概括
- **关键洞察**：多事件之间的关联发现
- **趋势判断**：事件背后的宏观趋势
- **风险预警**：潜在风险点识别
- **战略建议**：决策参考建议

## 注意事项

### 使用限制

1. **文本长度**：建议单次请求不超过10000字
2. **请求频率**：建议控制在5次/秒以内，避免触发LLM API限流
3. **超时设置**：建议客户端超时时间设置为30秒以上

### 最佳实践

1. **预处理文本**：
   - 去除HTML标签、特殊字符
   - 修正明显的错别字
   - 统一时间格式

2. **错误处理**：
   - 实现请求重试机制（最多3次）
   - 捕获超时异常
   - 记录失败日志

3. **结果验证**：
   - 检查 `events` 数组非空
   - 验证必填字段存在
   - 过滤明显不合理的事件

4. **性能优化**：
   - 对于超长文本，按段落拆分后分批调用
   - 缓存高频文本的处理结果
   - 使用异步调用避免阻塞主线程
