# 报文融合

多报文融合算法，将多篇报文的抽取结果合并后由 LLM 生成综合性融合报告。

## 文件说明

| 文件 | 说明 |
|------|------|
| fusion_extractor.py | FusionExtractor 类，四步融合流程：数据预处理 → 信息整合（合并时间线/实体/标签）→ LLM 生成（标题/摘要/内容）→ 结果组装 |
| requirements.txt | Python 依赖清单 |
| 需求文档.md | 融合功能需求文档 |

## 处理流程

1. **数据预处理**：统一各报文的字段格式
2. **信息整合**（代码实现）：合并时间线、合并实体（按标签归类）、合并标签（过滤"其他"）
3. **LLM 生成**：调用大模型生成标题、摘要和详细 Markdown 内容
4. **结果组装**：输出融合报告 JSON（含 title, summary, timeline, content, entities, labels）

## 依赖

- 导入 `数据抽取-新/llm_event_extractor.py` 中的 LLMEventExtractor 用于 LLM 调用
- 实体标签列表和 LLM 配置从 `数据抽取-新/config.json` 读取
