# 报文融合算法开发需求文档

## 1. 功能概述

### 1.1 目标
基于大语言模型（GLM-5），开发报文融合算法，将多篇（2-10篇）相关报文自动融合生成结构化的综合报告。

### 1.2 应用场景
- 用户在搜索结果页面选择多篇相关报文
- 点击"报文融合"按钮触发融合操作
- 系统生成包含标题、摘要、时间线、详细内容、关键实体、综合标签的融合报告
- 用户可保存、导出、编辑融合报告

### 1.3 核心价值
- 提高信息整合效率，减少人工整理时间
- 自动提取时间线脉络，便于事件追踪
- 综合关键实体信息，形成完整的情报画像
- 生成标准化报告格式，便于存档和分享

## 2. 输入输出规格

### 2.1 输入数据

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reportIds | List[Long] | 是 | 报文ID列表，长度范围 2-10 |
| fusionType | String | 否 | 融合类型：standard（标准）、timeline（时间线优先）、entity（实体优先），默认standard |
| customTitle | String | 否 | 用户自定义标题，若为空则自动生成 |

### 2.2 输出数据结构

```json
{
  "fusionId": Long,           // 融合报告唯一ID
  "title": String,            // 融合报告标题
  "summary": String,          // 报告摘要（200-500字）
  "timeline": [               // 事件时间线数组
    {
      "time": String,         // 时间点（格式：YYYY-MM-DD）
      "description": String   // 事件描述
    }
  ],
  "content": String,          // 详细内容（Markdown格式）
  "entities": {               // 关键实体
    "persons": [String],      // 人物列表
    "organizations": [String],// 组织机构列表
    "equipment": [String]     // 武器装备列表
  },
  "labels": [String],         // 综合标签列表
  "sourceIds": [Long],        // 来源报文ID列表
  "modelUsed": String,        // 使用的模型名称
  "createTime": String,       // 创建时间
  "updateTime": String        // 更新时间
}
```

## 6. 代码实现详解

### 6.1 时间线整合（代码实现）

```python
def merge_timeline(all_events: List[List[Event]]) -> List[Dict]:
    """
    用代码合并时间线，不调用LLM
    
    Args:
        all_events: 各报文的事件列表
    
    Returns:
        [
            {"time": "2024-03-15", "description": "美军B-52轰炸机执行南海巡航任务"},
            {"time": "2024-03-18", "description": "白宫签署国防预算行政命令"},
            ...
        ]
    
    实现步骤：
    1. 提取所有事件的time和action字段
    2. 标准化时间格式（YYYY-MM-DD）
    3. 按时间排序
    4. 合并相同时间点的多个事件（用分号连接描述）
    """
    timeline = []
    
    # 1. 收集所有时间-事件对
    time_events = {}
    for events in all_events:
        for event in events:
            time_str = normalize_time(event.get('time', ''))
            if time_str:
                if time_str not in time_events:
                    time_events[time_str] = []
                time_events[time_str].append(event.get('action', ''))
    
    # 2. 按时间排序
    sorted_times = sorted(time_events.keys())
    
    # 3. 构建时间线
    for t in sorted_times:
        description = '; '.join(time_events[t])
        timeline.append({"time": t, "description": description})
    
    return timeline
```

### 6.2 实体整合（代码实现）

```python
def merge_entities(all_entities: List[Entities]) -> Dict:
    """
    用代码合并实体，不调用LLM
    
    Args:
        all_entities: 各报文的实体信息
    
    Returns:
        {
            "persons": ["拜登", "白宫发言人", ...],
            "organizations": ["美军", "北约", "白宫", ...],
            "equipment": ["B-52轰炸机", "F-35战斗机", ...]
        }
    
    实现步骤：
    1. 收集所有人物，去重
    2. 收集所有组织，去重
    3. 收集所有武器装备，去重
    """
    persons = set()
    organizations = set()
    equipment = set()
    
    for entities in all_entities:
        if entities.get('persons'):
            persons.update(entities['persons'])
        if entities.get('organizations'):
            organizations.update(entities['organizations'])
        if entities.get('equipment'):
            equipment.update(entities['equipment'])
    
    return {
        "persons": list(persons),
        "organizations": list(organizations),
        "equipment": list(equipment)
    }
```

### 6.3 标签生成（代码实现）

```python
def generate_labels(entities: Dict, timeline: List[Dict]) -> List[str]:
    """
    用代码生成标签，不调用LLM
    
    Args:
        entities: 整合后的实体信息
        timeline: 整合后的时间线
    
    Returns:
        ["军事", "北约", "美国", "南海", "轰炸机", ...]
    
    实现步骤：
    1. 从实体中提取关键词作为标签
    2. 从事件描述中提取关键词
    3. 基于事件类型添加分类标签
    """
    labels = []
    
    # 1. 从实体提取（取前3个）
    labels.extend(entities['persons'][:3])
    labels.extend(entities['organizations'][:3])
    labels.extend(entities['equipment'][:2])
    
    # 2. 从事件描述提取关键词
    for item in timeline[:3]:
        keywords = extract_keywords(item['description'])
        labels.extend(keywords[:2])
    
    # 3. 去重
    labels = list(set(labels))
    
    # 4. 确保至少5个标签
    if len(labels) < 5:
        labels.extend(['政治', '国际', '军事', '外交'])
    
    return labels[:8]  # 最多8个标签
```

## 7. 简化后的算法流程

```
输入：reportIds
    │
    ▼
Step 1: 数据提取（代码）
    │ 获取报文 + 属性抽取结果
    │
    ▼
Step 2: 信息整合（代码）
    │ ├── 合并时间线 → timeline JSON
    │ ├── 合并实体   → entities JSON
    │ └── 生成标签   → labels数组
    │
    ▼
Step 3: LLM生成
    │ 输入：报文摘要 + 实体列表
    │ 输出：title + summary + content
    │
    ▼
Step 4: 结果组装（代码）
    │ 合并LLM输出 + 代码生成部分
    │
    ▼
输出：完整FusionDTO
```

**优化效果**：
- LLM只需生成3个字段（title、summary、content）
- 其他字段全部由代码处理，节省成本和时间
- Prompt大幅简化，降低LLM理解难度

## 10. 错误处理

### 7.1 错误类型

| 错误码 | 错误描述 | 处理方式 |
|--------|----------|----------|
| E001 | 报文数量不足（<2篇） | 返回错误提示 |
| E002 | 报文数量过多（>10篇） | 返回错误提示 |
| E003 | 报文ID不存在 | 返回错误提示，列出无效ID |
| E004 | 报文内容为空 | 跳过空报文，继续处理 |
| E005 | LLM调用失败 | 重试3次，失败后返回错误 |
| E006 | LLM返回格式错误 | 尝试修复JSON，失败后返回错误 |
| E007 | 处理超时 | 返回超时提示，建议减少报文数量 |

### 7.2 错误响应格式

```json
{
  "code": 0,
  "data": null,
  "msg": "E003: 报文ID不存在，无效ID: [100, 200]",
  "flag": "false"
}
```
