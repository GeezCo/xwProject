# 数据生成工具使用手册

## 工具概述

基于LLM的文本数据生成器，通过仿写样例文本批量生成相似风格的文本数据。

**核心能力：**
- 候选池随机化：时间、地点、人物、组织、装备、行为从候选池随机选择
- 日期范围生成：支持指定开始/结束日期，每个日期生成指定数量数据
- 多样性控制：条目数量和内容丰富度可独立调节
- 占位符补全：自动替换"某"、"XX"等占位符为具体内容

---

## 快速使用

### 基本命令

```bash
python data_generator.py <样例目录> <输出目录> [参数]
```

### 常用示例

```bash

#常用完全体指令
python data_generator.py 样例 输出_20260419 --start_date 2026-04-09 --end_date 2026-04-19 --total 1 --output_format jsonl --item_diversity low  --content_diversity high --model Qwen3.5-122B-A10B  --parallel 1 

# 生成今天的数据（每个样例10条）
python data_generator.py 样例 输出

# 生成指定日期的数据（每个样例100条）
python data_generator.py 样例 输出 --start_date 2026-04-19 --end_date 2026-04-19 --total 100

# 生成日期范围内的数据（jsonl汇总格式）
python data_generator.py 样例 输出 --start_date 2026-04-15 --end_date 2026-04-19 --total 100 --output_format jsonl

# 使用自定义模型
python data_generator.py 样例 输出  	 --total 100 --output_format jsonl
```

## 参数说明

### 必需参数

| 参数 | 说明 |
|------|------|
| `样例目录` | 包含样例txt文件的目录路径 |
| `输出目录` | 生成结果的保存目录 |

### 可选参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--total` | 10 | 每个样例、每个日期生成的数量 |
| `--parallel` | 3 | 并行线程数 |
| `--start_date` | 今天 | 开始日期（格式：YYYY-MM-DD） |
| `--end_date` | 今天 | 结束日期（格式：YYYY-MM-DD） |
| `--output_format` | txt | 输出格式（txt/jsonl） |
| `--model` | GLM-5 | 模型名称 |
| `--base_url` | https://llmapi.paratera.com | API地址 |
| `--item_diversity` | medium | 条目多样性（low/medium/high） |
| `--content_diversity` | medium | 内容多样性（low/medium/high） |

## 输出格式

### txt格式（默认）

每条数据保存为单独的txt文件：

```
输出/
├── 样例1/
│   ├── 标题1.txt
│   ├── 标题2.txt
│   └── ...
├── 样例2/
│   └── ...
└── generation_report.json
```

### jsonl格式（推荐）

所有数据汇总在一个jsonl文件中：

```
输出/
├── data_20260415_20260419.jsonl
└── generation_report.json
```

**jsonl数据结构：**
```json
{"时间": "2026年04月15日", "标题": "美军加强印太防御部署", "内容": "4月15日情况..."}
{"时间": "2026年04月16日", "标题": "多国军事动向报告", "内容": "4月16日..."}
```

## 多样性配置

### 条目多样性 (`--item_diversity`)

控制事件条目增加数量：

| 级别 | 条目增加范围 | 适用场景 |
|------|-------------|----------|
| low | 0-1个 | 保持样例结构，生成标准化报告 |
| medium | 1-3个 | 适度丰富内容 |
| high | 2-5个 | 生成详细情报通报 |

### 内容多样性 (`--content_diversity`)

控制候选池大小和生成随机性：

| 级别 | 温度 | 候选池大小 | 适用场景 |
|------|------|-----------|----------|
| low | 0.6 | 小 | 保守生成，贴近样例 |
| medium | 0.8 | 中 | 平衡多样性和一致性 |
| high | 0.95 | 大 | 高度随机，内容丰富 |

### 推荐组合

```bash
# 标准化报告生成（条目保守 + 内容丰富）
--item_diversity low --content_diversity high

# 详细情报生成（条目丰富 + 内容丰富）
--item_diversity high --content_diversity high
```

## 数据量计算

**公式：** 总数据量 = `日期天数 × 样例数 × total`

**示例计算：**

| 配置 | 计算 | 总量 |
|------|------|------|
| 默认配置 | 1天 × 6样例 × 10条 | 60条 |
| 单日期100条 | 1天 × 6样例 × 100条 | 600条 |
| 日期范围5天 | 5天 × 6样例 × 100条 | 3000条 |

## 样例文件格式

样例文件为普通txt文件，可包含占位符：

```
【标题】某国总统访问某地

某年某月某日，某国总统在某地与某人举行会晤。
双方就某问题进行了深入讨论...
```

**占位符自动替换：**
- `某国` → 美国中国日本等
- `某地` → 南海东海关岛等
- `某人` → 拜登习近平岸田等
- `某年某月某日` → 指定日期
- `某型飞机` → F-35B-1B等
- `某号舰` → 里根号林肯号等

## 注意事项

1. **生成速度**：取决于并行数和网络延迟，建议 `--parallel 1-3`
2. **数据保存**：生成过程中及时保存，中断不会丢失已完成部分
3. **标题处理**：自动清理非法字符，过长标题截断至50字
4. **重复处理**：相同标题自动添加序号后缀
5. **日期格式**：支持 YYYY-MM-DD、YYYYMMDD、YYYY/MM/DD

## 常见问题

### Q: 如何只生成今天的数据？

```bash
python data_generator.py 样例 输出
# 默认 start_date 和 end_date 都是今天
```

### Q: 如何生成过去7天的历史数据？

```bash
python data_generator.py 样例 输出 --start_date 2026-04-12 --end_date 2026-04-19 --output_format jsonl
```

### Q: jsonl和txt格式有什么区别？

| 格式 | 优点 | 适用场景 |
|------|------|----------|
| txt | 每条独立文件，便于查看 | 人工审阅、单独使用 |
| jsonl | 统一汇总，便于处理 | 数据分析、批量导入 |

### Q: 如何自定义候选池内容？

编辑 `candidate_pools.py` 文件，修改对应候选池列表：
- `TimePool` - 时间候选
- `LocationPool` - 地点候选
- `PersonPool` - 人物候选
- `OrganizationPool` - 组织候选
- `EquipmentPool` - 装备候选
- `ActionPool` - 行为候选
