# 算法模块

基于 Python 的 LLM 处理核心，提供事件抽取、报文融合、数据生成等算法能力。

## 子目录

| 目录 | 说明 |
|------|------|
| 算法服务/ | Flask REST API 服务（端口 5001），为后端提供 HTTP 接口 |
| 数据抽取-新/ | **当前生产版**事件抽取算法，支持多标签分类（从 config.json 读取标签列表） |
| 数据抽取-带标签/ | 旧版带标签抽取（使用 RexUniNlu 外部分类服务），保留作为参考 |
| 数据抽取/ | 最早版事件抽取算法（无标签分类），保留作为参考 |
| 报文融合/ | 多报文融合算法，合并时间线/实体后由 LLM 生成综合报告 |
| 数据生成/ | 基于样例的 LLM 仿写生成器，批量生成训练数据 |
| 事件拆分/ | 将多事件文本拆分为原子化独立事件 |

## 文件说明

| 文件 | 说明 |
|------|------|
| CLAUDE.md | Claude Code 项目指导文件 |
| LLM_api.txt | LLM API 地址和密钥配置参考 |

## 模块依赖关系

```
算法服务/app.py
    ├── 数据抽取-新/hierarchical_extractor.py  （事件抽取）
    │       └── 数据抽取-新/llm_event_extractor.py
    └── 报文融合/fusion_extractor.py           （报文融合）
            └── 数据抽取-新/llm_event_extractor.py
```

LLM 配置统一在 `数据抽取-新/config.json`，所有模块共用。
