## 二、核心问题及解决方案

### 问题1: 配置浅合并导致分类未生效

**问题描述**:  
修改 `config.json` 将 `subject_categories` 从 11 个分类改为 3 个后，重启算法服务仍识别出 11 个旧分类。

**根本原因**:  
`llm_event_extractor.py` 的 `load_config()` 函数使用 `dict.update()` 进行浅合并：

```python
# 旧代码（Line 84-85）
if key in default_config and isinstance(default_config[key], dict):
    default_config[key].update(user_config[key])  # ❌ 浅合并
```

对于嵌套字典 `subject_categories`，`update()` 会将用户配置的 3 个分类**追加**到默认的 8 个分类上，而非替换。

**解决方案**:  
对 `subject_categories` 字段进行完全替换：

```python
# 新代码（Line 84-88）
if key == 'subject_categories':
    # subject_categories 完全替换，不合并
    default_config[key] = user_config[key]
elif key in default_config and isinstance(default_config[key], dict):
    default_config[key].update(user_config[key])
```

**修改文件**: `算法/数据抽取-新/llm_event_extractor.py`

### 问题2: Python 模块缓存未清除

**问题描述**:  
修改代码后重启算法服务，配置仍未生效。

**根本原因**:  
- `LLMEventExtractor` 类的 `SUBJECT_CATEGORIES` 是类级别变量（Line 181），在模块首次导入时执行
- Python 解释器缓存了 `.pyc` 字节码文件和 `__pycache__` 目录
- 重启 Flask 服务不会清除这些缓存

**解决方案**:  
```bash
# 1. 强制终止所有 Python 进程
taskkill //F //IM python.exe

# 2. 清除 Python 缓存
find "算法" -type d -name "__pycache__" -exec rm -rf {} +
find "算法" -name "*.pyc" -delete

# 3. 重启算法服务
cd 算法/算法服务
python app.py
```
