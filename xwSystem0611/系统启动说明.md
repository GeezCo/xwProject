# 献微系统启动说明

## 系统架构

```
┌─────────────┐     HTTP      ┌─────────────┐     HTTP      ┌─────────────┐
│   前端      │ ───────────▶ │   后端      │ ───────────▶ │  算法服务   │
│  Vue 2      │              │ Spring Boot │              │   Flask     │
│  :8080      │              │   :8081     │              │   :5001     │
└─────────────┘              └─────────────┘              └─────────────┘
                                   │                              │
                                   ▼                              ▼
                             ┌─────────────┐              ┌─────────────┐
                             │   MySQL     │              │   LLM API   │
                             │   :8010     │              │   GLM-5     │
                             └─────────────┘              └─────────────┘
```

## 启动顺序

**重要：必须按以下顺序启动**

1. **算法服务** (端口5001) - 必须最先启动
2. **后端服务** (端口8081) - 依赖算法服务
3. **前端服务** (端口8080) - 依赖后端服务

## 一、算法服务启动

### 方式1：直接运行Python

```bash
cd D:\项目开发\献微系统\算法\算法服务
python app.py
```

### 方式2：后台运行（推荐）

```bash
cd D:\项目开发\献微系统\算法\算法服务

```

## 四、停止所有服务

### 查找端口进程

```bash
netstat -ano | findstr ":8080 :8081 :5001" | findstr LISTENING
```

### 停止进程

```bash
taskkill //F //PID <进程ID>
```

### 一键停止所有服务（示例）

```bash
# 查找并停止5001端口
netstat -ano | findstr ":5001" | findstr LISTENING
taskkill //F //PID <对应的PID>

# 查找并停止8081端口

taskkill //F //PID <对应的PID>

# 查找并停止8080端口
netstat -ano | findstr ":8080" | findstr LISTENING
taskkill //F //PID <对应的PID>
```

## 五、常见问题

### 1. 算法服务端口被占用

**现象**：`Address already in use: 5001`

**解决**：
```bash
# 查找占用5001端口的进程
netstat -ano | findstr ":5001"
# 停止进程
taskkill //F //PID <PID>
```

### 2. 后端无法连接算法服务

**现象**：后端调用算法接口超时

**原因**：算法服务未启动

**解决**：先启动算法服务，再启动后端

### 3. 前端无法访问后端API

**现象**：前端报错 `Network Error`

**原因**：后端未启动或CORS配置问题

**解决**：检查后端是否正常运行在8081端口

### 4. Maven Wrapper找不到spring-boot插件

**现象**：`No plugin found for prefix 'spring-boot'`

**原因**：未在后端目录下执行命令

**解决**：
```bash
# 必须切换到后端目录
cd D:\项目开发\献微系统\后端
./mvnw.cmd spring-boot:run
```
