# 献微系统启动说明

## 系统架构

```
┌─────────────┐     HTTP      ┌─────────────┐     HTTP      ┌─────────────┐
│   前端      │ ───────────▶ │   后端      │ ───────────▶ │  算法服务   │
│  Vue 3      │              │ Spring Boot │              │   Flask     │
│  :8080      │              │   :8081     │              │   :5001     │
└─────────────┘              └─────┬───────┘              └─────────────┘
                                   │
        ┌──────────────────────────┼─────────────────────────┐
        ▼                          ▼                         ▼
  ┌───────────┐            ┌───────────────┐         ┌───────────────┐
  │   MySQL   │            │ Elasticsearch │         │ BGE Embedding │
  │   :8010   │            │     :9200     │         │     :5002     │
  └───────────┘            └───────────────┘         └───────────────┘
```

## 启动顺序

**重要：必须按以下顺序启动**

1. **Elasticsearch** (端口9200) - 向量存储基础设施
2. **BGE Embedding 服务** (端口5002) - RAG 向量化服务
3. **算法服务** (端口5001) - LLM 抽取/融合
4. **后端服务** (端口8081) - 依赖以上服务
5. **前端服务** (端口8080) - 依赖后端服务

> **说明**：步骤 1-2 仅在使用 RAG 知识库功能时需要启动；只用抽取/融合功能可跳过。

## 〇、Elasticsearch 启动（RAG 功能依赖）

```bash

```

首次部署需安装 IK 分词器：

```bash

unzip elasticsearch-analysis-ik-7.17.15.zip -d ${ES_HOME}/plugins/ik
```


## 四、停止所有服务

### 查找端口进程

```bash
netstat -ano | findstr ":8080 :8081 :5001" | findstr LISTENING
```

### 停止进程


## 五、常见问题

### 1. 算法服务端口被占用

**现象**：`Address already in use: 5001`

**解决**：
```bash
# 查找占用5001端口的进程
netstat -ano | findstr ":5001"
# 停止进程
taskkill //F //PID <PID>
```

### 2. 后端无法连接算法服务

**现象**：后端调用算法接口超时

**原因**：算法服务未启动

**解决**：先启动算法服务，再启动后端

### 3. 前端无法访问后端API

**现象**：前端报错 `Network Error`

**原因**：后端未启动或CORS配置问题

**解决**：检查后端是否正常运行在8081端口

### 4. Maven Wrapper找不到spring-boot插件

**现象**：`No plugin found for prefix 'spring-boot'`

**原因**：未在后端目录下执行命令

**解决**：
```bash
# 必须切换到后端目录
cd D:\项目开发\献微系统\后端
./mvnw.cmd spring-boot:run
```
