## 测试步骤

### 1. 启动系统

确保三个服务已启动：
- 算法服务：http://localhost:5001
- 后端服务：http://localhost:8081
- 前端服务：http://localhost:8080

### 2. 登录系统

访问 http://localhost:8080，使用账号登录。

### 3. 进入数据看板

点击顶部导航栏"数据看板"按钮。

### 4. 导入测试数据

在"数据库管理"面板中：

1. **选择分类**：
   - 一级分类：选择任意一级分类（如"开源信息"）
   - 二级分类：选择对应的二级分类（如"开源获情"）

2. **选择 JSONL 文件**：
   - 点击"选择 JSONL"按钮
   - 浏览到本目录，选择 `test_data.jsonl`

3. **选择图片文件**：
   - 点击"选择图片"按钮
   - 按住 Ctrl 键，选择所有 6 张图片（exercise_1.jpg ~ russia_navy_3.jpg）
   - 或者直接全选本目录下的所有 .jpg 文件

4. **开始导入**：
   - 点击"开始导入"按钮
   - 等待导入完成（约 5-10 秒）

### 5. 验证结果

导入完成后，检查导入结果：

**预期结果**：
```
成功：4 条
失败：0 条
总行数：4
已上传图片：6 张
```

**数据验证**：
1. 进入"报文列表"页面
2. 筛选刚才选择的分类
3. 应该看到 4 条新报文：
   - 3 条图文报（带有图片图标）
   - 1 条文字报

4. 点击图文报的"查看详情"按钮
5. 详情面板应该显示图片网格
6. 图片应该正常加载显示

## 常见问题

### Q1: 导入失败，提示"一级分类不存在"

**解决**：请先在"分类管理"中创建对应的分类，或选择系统中已有的分类。

### Q2: 导入成功，但图片未显示

**可能原因**：
1. MinIO 服务未启动或配置错误
2. 网络无法访问 MinIO 地址
3. 图片文件名与 JSONL 中不匹配（区分大小写）

**排查方法**：
1. 检查后端日志，查看 MinIO 上传是否成功
2. 检查数据库 `origin_text` 表的 `images` 字段内容
3. 手动访问 MinIO 控制台（通常是 http://localhost:9000）查看文件

### Q3: 部分图片上传失败

**查看日志**：后端日志会显示"图片上传失败: xxx.jpg"

**常见原因**：
- 图片文件损坏
- 图片格式不支持
- MinIO 存储空间不足
