# 前端升级：图文报图片展示 + 等待名言效果

## 背景

当前系统有三种报文模态（文字报/图文报/声像报），但前端对图文报没有图片展示能力，详情面板只显示纯文本。此外，属性抽取（2-5分钟）和报文融合（5-8分钟）等待时间长，用户体验差——当前仅有一个旋转图标+静态文字提示。

本次升级：
1. 数据库新增图片字段，前端详情面板支持图文报图片展示
2. 抽取/融合等待过程中，随机轮播军事/情报主题名人名言

## 功能一：图文报图片展示

### 1.1 图片存储方案 — 后端外部目录

**图片文件**存放在服务器固定目录 `D:/data/xianwei-images/`，文件命名规则：`{sid}_{序号}.jpg`（如 `188_1.jpg`, `188_2.jpg`）

**Spring Boot 静态资源映射**：

**文件**: [后端/src/main/resources/application.yml](后端/src/main/resources/application.yml)

新增配置，将外部目录映射为 `/images/` HTTP路径：
```yaml
spring:
  web:
    resources:
      static-locations:
        - classpath:/static/
        - file:D:/data/xianwei-images/
```

同时新增 **WebMvcConfigurer** 做路径映射（更精确控制）：

**文件**: `后端/src/main/java/com/qy/dch/config/WebMvcConfig.java`（新建）

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/images/**")
                .addResourceLocations("file:D:/data/xianwei-images/");
    }
}
```

访问方式：`http://localhost:8081/images/188_1.jpg`

### 1.2 数据库 — 新增 images 字段

**文件**: `后端/src/main/resources/db/origin_text_images_ddl.sql`（新建）

```sql
ALTER TABLE origin_text
ADD COLUMN images TEXT DEFAULT NULL COMMENT '图片文件名列表（JSON数组格式，如 ["188_1.jpg","188_2.jpg"]）';
```

数据库只存文件名，前端拼接完整URL。

### 1.3 后端 DTO — 新增 images 字段

**文件**: [后端/src/main/java/com/qy/dch/dto/OriginTextDTO.java](后端/src/main/java/com/qy/dch/dto/OriginTextDTO.java)

```java
/** 图片文件名列表（JSON数组字符串） */
private String images;
```

### 1.4 后端 Mapper — 查询语句加 images

**文件**: [后端/src/main/java/com/qy/dch/mapper/UygurMapper.java](后端/src/main/java/com/qy/dch/mapper/UygurMapper.java)

需要更新以下 SQL 查询（共2处关键查询）：
- `getDetailBySid`（约270行）: SELECT 加上 `images` 字段
- `getTextListByTypeIdsAndModalTypesPaged`（约210行）: SELECT 加上 `images` 字段

其他分页查询也需同步加 images，但按实际使用频率优先处理这两个。

### 1.5 前端详情面板 — 图片展示区

**文件**: [前端/qyportrait/src/components/searchResult.vue](前端/qyportrait/src/components/searchResult.vue)

在详情面板的 `detail-text` 上方，`detail-meta` 下方，插入图片展示区域：

```html
<!-- 图片展示（图文报） -->
<div class="detail-images" v-if="detailImages.length">
  <div class="img-grid">
    <div v-for="(url, idx) in detailImages" :key="idx" class="img-item" @click="previewImage(url)">
      <img :src="url" :alt="'图片' + (idx+1)" loading="lazy" />
    </div>
  </div>
</div>
```

逻辑：
- `detailImages` 计算属性：解析 `activeDetail.images`（JSON字符串 → 文件名数组），拼接后端URL前缀 `http://localhost:8081/images/`，无图片则为空数组
- `previewImage(url)`: 使用 Element Plus 的 `ElImageViewer` 或简单的全屏预览
- 图片网格布局：单图占满宽度，2-4图用2列网格，图片等比缩放

样式要点：
```css
.detail-images { margin: 12px 0; }
.img-grid { display: grid; grid-template-columns: repeat(2, 1fr); gap: 8px; }
.img-grid:has(> :only-child) { grid-template-columns: 1fr; } /* 单图占满 */
.img-item img { width: 100%; border-radius: 6px; cursor: zoom-in; object-fit: cover; max-height: 200px; }
```

### 1.5 中间列表项 — 图文报缩略图标记

**文件**: [前端/qyportrait/src/components/searchResult.vue](前端/qyportrait/src/components/searchResult.vue)

在列表项 `.item-title` 旁或 `.item-time` 行，给图文报加一个小图标标记（用文字或 icon），让用户在列表里就能区分文字报和图文报：

```html
<span class="modal-badge" v-if="item.modalType === '图文报'">图</span>
```

### 2.2 searchResult.vue — 抽取等待名言

**文件**: [前端/qyportrait/src/components/searchResult.vue](前端/qyportrait/src/components/searchResult.vue)

**当前代码**（第160-163行）：
```html
<div v-if="extracting" style="text-align:center;padding:20px;color:var(--color-text-secondary);">
  <el-icon class="is-loading" :size="20"><Loading /></el-icon>
  <div style="margin-top:8px;">抽取中，预计2-5分钟...</div>
</div>
```

**替换为**：
```html
<div v-if="extracting" class="extracting-panel">
  <el-icon class="is-loading" :size="24"><Loading /></el-icon>
  <div class="extracting-status">属性抽取中，预计2-5分钟...</div>
  <transition name="quote-fade" mode="out-in">
    <div class="quote-card" :key="currentQuote.text">
      <div class="quote-text">"{{ currentQuote.text }}"</div>
      <div class="quote-author">—— {{ currentQuote.author }}
        <span v-if="currentQuote.source"> {{ currentQuote.source }}</span>
      </div>
    </div>
  </transition>
</div>
```

JS逻辑：
```js
import { QUOTES } from '@/constants/quotes'

const currentQuote = ref(QUOTES[Math.floor(Math.random() * QUOTES.length)])
let quoteTimer = null

watch(extracting, (val) => {
  if (val) {
    currentQuote.value = QUOTES[Math.floor(Math.random() * QUOTES.length)]
    quoteTimer = setInterval(() => {
      currentQuote.value = QUOTES[Math.floor(Math.random() * QUOTES.length)]
    }, 8000)
  } else {
    clearInterval(quoteTimer)
  }
})
```

### 2.3 fusionResult.vue — 融合等待名言

**文件**: [前端/qyportrait/src/components/fusionResult.vue](前端/qyportrait/src/components/fusionResult.vue)

**当前**（第29行）使用 Element Plus 的 `v-loading` 指令：
```html
<div class="report-panel" v-loading="loading" element-loading-text="正在融合报文，预计5-8分钟...">
```

```html
<div class="report-panel">

  <div v-if="loading" class="fusion-loading">

      <div class="quote-card" :key="currentQuote.text">
        <div class="quote-text">"{{ currentQuote.text }}"</div>
        <div class="quote-author">—— {{ currentQuote.author }}
          <span v-if="currentQuote.source"> {{ currentQuote.source }}</span>
        </div>
      </div>
    </transition>
  </div>
  <!-- 融合结果（原有内容） -->
  <template v-if="fusionReport && !loading">
    ...
  </template>
</div>
```

JS逻辑同 searchResult.vue，使用相同的 QUOTES 导入和 8秒轮换 interval。
