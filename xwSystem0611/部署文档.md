# 献微系统部署文档

## 一、环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 8 | 后端运行时（全局统一） |
| Maven | 3.8+ | 后端构建 |
| Node.js | 14+ | 前端构建 |
| Python | 3.10+ | 算法服务 / Embedding 服务 |
| MySQL | 8.0 | 数据库 |
| Elasticsearch | 7.17.15 | RAG 向量存储（含 IK 分词器） |

---

## 二、服务端口

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端 | 8080 | Vue 开发服务器 |
| 后端 | 8081 | Spring Boot |
| 算法服务 | 5001 | Flask（LLM 抽取/融合） |
| BGE Embedding | 5002 | Flask（RAG 向量化） |

| Druid监控 | 8081/druid | 数据库连接池监控面板 |

## 三、启动顺序

**必须按以下顺序启动**：ES → BGE Embedding → 算法服务 → 后端 → 前端

> 注：仅在使用 RAG 知识库功能时需要启动 ES 和 BGE Embedding 服务### 3.0 启动 Elasticsearch（RAG 依赖）

```bash
cd /path/to/elasticsearch-7.17.15
./bin/elasticsearch -d
```

验证：
```bash
curl http://localhost:9200
```

### 3.0.5 启动 BGE Embedding 服务（RAG 依赖）

```bash
cd xwEmbedding
pip install -r requirements.txt
python app.py
```

验证：
```bash
curl http://localhost:5002/health
# 预期：{"status":"ok","model":"bge-large-zh-v1.5","dimension":1024}
```

### 3.1 启动算法服务

```bash
cd 算法/算法服务

# 安装依赖（首次）
pip install flask flask-cors openai requests

# 启动服务
python app.py
```

验证：
```bash
curl http://localhost:5001/health
# 预期：{"status": "ok", "service": "algorithm-service", ...}
```

### 3.2 启动后端服务

```bash
cd 后端

# 方式1：Maven启动
mvn spring-boot:run

# 方式2：打包后运行
mvn clean package -DskipTests
java -jar target/dch-0.0.1-SNAPSHOT.jar
```

验证：
```bash
curl http://localhost:8081/uygur/category
# 预期：{"code": 1, "msg": "success", "data": [...]}
```

### 3.3 启动前端服务

```bash
cd 前端/qyportrait

# 安装依赖（首次）
npm install

# 启动开发服务器
npm run dev
```

验证：浏览器访问 `http://localhost:8080`，看到登录页面

### 4.2 初始化

首次部署需执行DDL脚本（按顺序）：

```bash

mysql -h 36.103.234.242 -P 8010 -u root -p uygur_project < 后端/src/main/resources/db/extraction_ddl.sql

# 2. 创建融合报告表

# 3. 添加模态类型字段
mysql -h 36.103.234.242 -P 8010 -u root -p uygur_project < 后端/src/main/resources/db/origin_text_modal_type_ddl.sql
```

### 4.3 数据导入

首次需导入报文数据：
1. 将数据文件（JSON格式）放到后端配置的 `filePath` 路径
2. 访问 `GET http://localhost:8081/uygur/savetext` 触发导入

## 七、常见问题

### 7.1 算法服务端口被占用

```bash
# Windows: 查找占用5001端口的进程
netstat -ano | findstr :5001

# 终止进程
taskkill /F /PID <PID>
```

### 7.2 抽取或融合超时

- 抽取操作耗时2-5分钟，融合操作耗时5-8分钟，属于正常现象
- 如果超过10分钟无响应，检查算法服务日志：`算法/算法服务/logs/algorithm.log`
- 确认LLM API是否可达：`curl https://llmapi.paratera.com/v1/models -H "Authorization: Bearer <api_key>"`

### 7.3 数据库连接失败

- 确认数据库服务器可达：`telnet 36.103.234.242 8010`
- 检查 `application.yml` 中的用户名密码
- 查看 Druid 监控面板：`http://localhost:8081/druid`（admin/admin）

### 7.4 前端请求后端跨域

后端已通过 `CorsConfig.java` 配置全局CORS，允许所有来源。如仍有跨域问题，检查前端 `vue.config.js` 的代理配置。
