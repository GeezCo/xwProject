# 献微系统 — 功能完善实现计划

## Context

在现有登录/搜索/抽取/融合功能基础上，需要完成三大块改动：
1. **搜索结果页优化**：左侧分类目录扁平化 + 新增"目标名称搜索融合"入口
2. **数据统计看板**：新建 Dashboard.vue 页面，含核心指标、图表、融合报告列表
3. **批量属性抽取**：支持按时间段批量抽取未抽取报文，前端可视化进度

## 三、批量属性抽取

### 3.1 后端接口

**新增到 `controller/ExtractionController.java`：**

| 接口 | 方法 | 功能 |
|------|------|------|
| `POST /extraction/batch/start` | POST | 启动批量抽取任务。参数: `{startDate, endDate, scope: "unextracted"\|"all"}`。返回: `{taskId, totalCount}` |
| `GET /extraction/batch/progress/{taskId}` | GET | 查询批量抽取进度。返回: `{taskId, total, done, failed, running, currentSid, logs[]}` |
| `POST /extraction/batch/stop/{taskId}` | POST | 停止批量抽取任务 |

**新增 `service/impl/ExtractionServiceImpl.java` 方法：**
- `startBatchExtraction(startDate, endDate, scope)`:
  1. 查询时间段内符合条件的报文 ID 列表
  2. 创建异步任务（`@Async` 或线程池），逐篇调用已有的 `extract(originTextId, force)` 方法
  3. 用 `ConcurrentHashMap<String, BatchTaskState>` 保存任务状态
  4. 返回 taskId + totalCount
- `getBatchProgress(taskId)`: 从 ConcurrentHashMap 读取当前状态
- `stopBatchTask(taskId)`: 设置停止标志，任务循环检测后中止

**扩展 Mapper：**
- `UygurMapper` 新增: `selectIdsByTimeRange(startDate, endDate, isExtracted)` — 按时间段查询报文 sid 列表

### 3.2 前端（集成在 Dashboard.vue 中）

批量抽取面板（参照 HTML 线框图的"批量属性抽取"区域）：
- 时间范围选择器（el-date-picker daterange）
- 抽取范围单选（仅未抽取 / 全部）
- 启动/停止按钮
- 实时进度条（百分比 + shimmer 动画）
- 4 个计数器：待抽取 / 已完成 / 失败 / 预计剩余时间
- 日志滚动面板

进度轮询：启动后每 3 秒轮询 `GET /extraction/batch/progress/{taskId}`，更新 UI 状态。

## 实现顺序

1. **后端基础接口**: TargetSearchRequest → ExtractionMapper 扩展 → FusionService.searchByTarget → FusionController
2. **后端统计接口**: UygurMapper 扩展 → DashboardService → DashboardController
3. **后端批量抽取**: UygurMapper.selectIdsByTimeRange → ExtractionService batch 方法 → ExtractionController batch 接口
4. **前端 searchResult.vue**: 分类扁平化 → 目标搜索 UI + 逻辑
5. **前端 fusionResult.vue + store**: fusionType 适配
6. **前端 Dashboard.vue**: 新建页面 → 统计展示 → 批量抽取面板
7. **前端 Home.vue + router**: 导航菜单 + 路由注册

## 验证方式

1. `curl POST /api/fusion/searchByTarget` 验证目标搜索返回匹配报文
2. `curl GET /api/dashboard/overview` 验证统计数据
3. `curl POST /extraction/batch/start` → `GET /extraction/batch/progress/{taskId}` 验证批量抽取流程
4. 前端 `npm run dev`:
   - 搜索页: 左侧分类平铺显示 → 目标搜索框输入名称 → 列表显示匹配报文 → 融合
   - 看板页: 菜单点击进入 → 指标卡片 + 图表正常 → 批量抽取启动/停止/进度
