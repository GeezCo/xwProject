## 交互逻辑

### 1. 地图交互
```
点击热力区域 → 右侧显示区域详情 → 高亮该区域
双击热力区域 → 放大到该区域 → 显示详细标记
右键热力区域 → 显示上下文菜单(查看事件/生成报告/添加监控)
悬停热力区域 → 显示tooltip(区域名称、事件数量、热度等级)
点击标记 → 显示事件详情弹窗
拖拽地图 → 平移视图
滚轮缩放 → 调整地图缩放级别
```

### 2. 筛选交互
```
勾选事件类型 → 实时更新热力图显示
选择热力指标 → 重新计算热力值并更新颜色
切换地图样式 → 更换底图
调整时间范围 → 过滤指定时间段的数据
搜索地点 → 定位到指定位置并高亮
```

### 3. 聚类展开
```
缩小地图 → 自动聚类相近区域的事件
聚类标记显示 → 数字表示包含的事件数量
点击聚类标记 → 放大并展开为单个事件标记
放大地图 → 自动展开聚类，显示详细标记
```

### 4. 时间动画
```
播放时间动画 → 按时间顺序显示事件出现
热力图动态变化 → 反映不同时间段的热度分布
标记逐个出现 → 模拟事件发生过程
暂停动画 → 停留在当前时间点
```

## 技术实现

### 1. 地图渲染库
- **推荐**: Leaflet + Leaflet.heat (轻量级，易于集成)
- **备选**: Mapbox GL JS (功能强大，视觉效果好)
- **备选**: 高德地图 API (国内地图数据准确)
- **备选**: ECharts Geo (与现有技术栈一致)

### 2. 热力图算法
```javascript
// 计算热力值
function calculateHeatValue(events, metric) {
  switch (metric) {
    case '事件数量':
      return events.length
    case '重要性':
      return events.reduce((sum, e) => {
        const weight = { '高': 3, '中': 2, '低': 1 }
        return sum + weight[e.importance]
      }, 0)
    case '活跃度':
      const now = Date.now()
      return events.reduce((sum, e) => {
        const age = (now - new Date(e.time)) / (1000 * 60 * 60 * 24) // 天数
        const decay = Math.exp(-age / 7) // 7天衰减
        const weight = { '高': 3, '中': 2, '低': 1 }
        return sum + weight[e.importance] * decay
      }, 0)
  }
}
```

### 3. 性能优化
- **数据聚合**: 服务端预聚合网格数据，减少传输量
- **分级加载**: 根据缩放级别加载不同精度的数据
- **Canvas渲染**: 使用Canvas渲染热力图，提升性能
- **防抖处理**: 拖拽和缩放操作使用防抖
