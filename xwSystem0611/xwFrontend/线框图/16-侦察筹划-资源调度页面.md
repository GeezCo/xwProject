## 布局说明

### 1. Tab 1 - 资源总览
- **状态概览卡片**: 各类资源可用/总数/任务中/维护中
- **详细列表**: 按类别展示每个资源的详细状态
  - 无人机: 编号、型号、状态、载荷、续航、位置、当前任务
  - 地面侦察: 编号、人数、装备、状态、位置
  - 技术侦察: 电子侦察站、卫星窗口
- **操作**: 分配到方案、送维护、查看任务

### 2. Tab 2 - 调度日历
- **甘特图视图**: 资源(行) × 时间(列)
- **已分配任务**: 色块标注，显示方案编号
- **维护时段**: 灰色色块
- **卫星窗口**: 时间点标注
- **冲突检测**: 自动提示资源冲突
- **建议**: 基于使用情况建议维护安排

### 3. Tab 3 - 任务监控
- **实时态势图**: 显示执行中任务的资源位置和航线
  - 无人机当前位置和飞行航线
  - 地面侦察班位置（定时上报）
  - 乌方(左)/俄方(右)标识
- **任务列表**: 执行中/待执行/草稿状态的任务
- **实时数据流**: 滚动显示侦察资源回传的实时信息

<template>
  <div class="recon-resource-page">
    <PageHeader title="侦察资源调度" />

    <a-tabs v-model:activeKey="activeTab">
      <a-tab-pane key="overview" tab="资源总览">
        <AssetSummaryCards :assets="assets" />
        <AssetDetailTable :assets="assets" @assign="assignAsset" @maintain="sendToMaintenance" />
      </a-tab-pane>

      <a-tab-pane key="calendar" tab="调度日历">
        <ScheduleCalendar
          :schedule="schedule"
          :conflicts="conflicts"
          :suggestions="suggestions"
          @assign="openAssignDialog"
          @reschedule="reschedule"
        />
      </a-tab-pane>

      <a-tab-pane key="monitor" tab="任务监控">
        <div class="monitor-layout">
          <MissionMap :missions="activeMissions" :assets="missionAssets" />
          <MissionList :missions="allMissions" @detail="showDetail" @recall="recallAsset" />
        </div>
        <DataFeed :feed="dataFeed" />
      </a-tab-pane>
    </a-tabs>
  </div>
</template>

<script setup>
import { ref, onMounted, onUnmounted } from 'vue'
import { useReconStore } from '@/stores/recon'
import { useWebSocket } from '@/composables/useWebSocket'

const reconStore = useReconStore()
const activeTab = ref('overview')

const { connect, disconnect } = useWebSocket('ws://api/recon/monitor', {
  onMessage: (data) => {
    if (data.type === 'mission_update') {
      reconStore.updateMission(data.data)
    }
  }
})

onMounted(() => {
  connect()
  reconStore.loadAssets()
  reconStore.loadSchedule()
  reconStore.loadActiveMissions()
})

onUnmounted(() => disconnect())
</script>
