# 前端修改流程

## 概述

本文档规定了献微系统前端新增或修改功能时的标准流程，确保需求、设计、实现的一致性。

---

## 标准流程

### 1. 调整或新增线框图

**目标**: 将功能需求转化为可视化的页面结构和交互设计

**操作**:
- 在 `前端/线框图/` 目录下创建或修改 Markdown 文件
- 使用文字描述页面布局、组件层次、交互逻辑
- 明确标注：
  - 页面区域划分（顶栏/侧栏/主体/底栏）
  - 组件类型（输入框/按钮/表格/卡片等）
  - 数据流向（接口调用/状态管理）
  - 交互行为（点击/筛选/分页等）

**示例**:
```
## 03-搜索结果页面

### 布局
- 三栏布局：目录(220px) | 列表(flex:1) | 详情(340px)

### 目录面板
- 标题："信息来源分类"
- 分类树：一级分类 + 二级分类（可折叠）
- 点击分类 → 筛选列表

### 列表面板
- 搜索栏：输入框 + 搜索按钮
- 筛选栏：排序/来源/标签/时间
- 报文卡片：标题/时间/标签/内容预览
- 勾选框：支持多选（2-10篇）
- 融合按钮：已选N篇 → 跳转融合页
```

---

### 2. 根据用户意见调整线框图

**目标**: 与用户确认需求理解无误

**操作**:
- 将线框图提交给用户审阅
- 收集反馈：布局是否合理、交互是否符合预期、是否遗漏功能点
- 根据反馈修改线框图，直到用户确认通过

**注意**:
- 此阶段不涉及代码，修改成本低
- 重点确认：页面结构、数据展示方式、用户操作流程

---

### 3. 基于线框图生成 HTML

**目标**: 将线框图转化为带样式的静态 HTML 原型

**操作**:
- 在 `前端/线框图HTML/` 目录下创建 HTML 文件
- 使用内联 CSS 实现完整样式（颜色/字号/间距/布局）
- 使用 CSS 变量统一主题（暗色/亮色双主题）
- 填充模拟数据，展示真实效果

**要求**:
- 像素级还原线框图设计
- 响应式布局（适配不同屏幕）
- 交互状态完整（hover/active/disabled）
- 双主题支持（通过 `data-theme` 属性切换）

**示例**:
```html
<!DOCTYPE html>
<html data-theme="dark">
<head>
  <style>
    :root, [data-theme="dark"] {
      --bg-page: #0a1628;
      --color-primary: #009afb;
      --color-title: #fba700;
    }
    [data-theme="light"] {
      --bg-page: #eef1f6;
      --color-primary: #1a73e8;
      --color-title: #d48806;
    }
    body { background: var(--bg-page); }
    .title { color: var(--color-title); font-size: 17px; }
  </style>
</head>
<body>
  <div class="search-page">
    <div class="category-panel">...</div>
    <div class="list-panel">...</div>
    <div class="detail-panel">...</div>
  </div>
</body>
</html>
```

---

### 4. 根据用户意见调整 HTML

**目标**: 确认视觉效果和交互细节

**操作**:
- 在浏览器中打开 HTML 文件，展示给用户
- 收集反馈：颜色/字号/间距/布局/交互效果
- 调整 CSS 样式，直到用户满意

**注意**:
- 此阶段仍是静态页面，无真实数据和接口调用
- 重点确认：视觉风格、信息密度、操作便捷性

---

### 5. 根据调整后 HTML 实现前端代码

**目标**: 将 HTML 原型转化为 Vue 3 组件，接入真实接口

**操作**:

#### 5.1 创建 Vue 组件
- 在 `前端/qyportrait/src/components/` 或 `src/views/` 下创建 `.vue` 文件
- 使用 `<script setup>` 语法（Vue 3 Composition API）
- 从 HTML 提取结构到 `<template>`，提取样式到 `<style scoped>`

#### 5.2 接入接口
- 在 `src/utils/request.js` 中封装 API 调用
- 使用 `ref`/`reactive` 管理组件状态
- 使用 `onMounted` 等生命周期钩子加载数据

#### 5.3 状态管理
- 跨组件共享状态使用 Pinia（`src/stores/`）
- 路由跳转使用 Vue Router（`src/router/`）

#### 5.4 样式迁移
- 将 HTML 中的 CSS 变量复用到 `src/assets/theme.css`
- 组件内样式使用 `<style scoped>`，避免污染全局
- 保持与 HTML 原型的视觉一致性

#### 5.5 测试验证
- 启动前端 `npm run dev`
- 在浏览器中测试完整流程
- 确认接口调用正常、数据展示正确、交互符合预期

**示例**:
```vue
<template>
  <div class="search-page">
    <div class="category-panel">
      <div v-for="cat in categories" :key="cat.id" @click="selectCategory(cat.id)">
        {{ cat.typeName }}
      </div>
    </div>
    <div class="list-panel">
      <div v-for="item in reportList" :key="item.sid">
        {{ item.title }}
      </div>
    </div>
  </div>
</template>

<script setup>
import { ref, onMounted } from 'vue'
import request from '@/utils/request'

const categories = ref([])
const reportList = ref([])

onMounted(async () => {
  const res = await request.get('/uygur/category')
  categories.value = res.data
})

async function selectCategory(typeId) {
  const res = await request.post('/uygur/getTextList', { pageNum: 1, pageSize: 20, typeId })
  reportList.value = res.data.list
}
</script>

<style scoped>
.search-page { display: flex; gap: 16px; }
.category-panel { width: 220px; background: var(--bg-panel); }
.list-panel { flex: 1; }
</style>
```

---

## 流程图

```
需求 → 线框图(MD) → 用户确认 → HTML原型 → 用户确认 → Vue组件 → 测试验证
  ↓         ↓                        ↓                        ↓
调整     调整                      调整                    修复bug
```

---

## 注意事项

1. **不要跳过步骤**: 直接写代码容易偏离需求，返工成本高
2. **充分沟通**: 每个阶段都要与用户确认，避免理解偏差
3. **保持一致**: HTML 原型和最终代码的视觉效果应完全一致
4. **文档同步**: 修改功能时同步更新线框图和 HTML 文件
