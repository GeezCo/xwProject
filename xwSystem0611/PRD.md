## 1. 产品概述

### 1.1 产品名称

献微系统 (Xianwei System)

### 1.2 产品定位

基于大语言模型（LLM）的智能文本分析与事件抽取系统，专注于军事、政治、外交等领域文本信息的结构化处理。

### 1.3 目标用户

- 情报分析人员
- 数据分析研究员
- 信息整编工作人员
- 军事动态监测人员

### 1.4 核心价值

- **自动化处理**：将非结构化文本自动转化为结构化事件数据
- **降低成本**：层次化抽取算法节省约50%的LLM调用成本
- **提高效率**：从人工阅读整编转变为机器自动抽取
- **知识沉淀**：抽取结果持久化存储，支持后续分析

### 2.2 属性抽取模块

#### 2.2.1 功能描述

基于大语言模型的智能事件抽取，从文本中自动识别并提取事件要素。

#### 2.2.2 抽取要素

| 要素 | 描述 | 示例 |
|------|------|------|
| time | 事件发生时间 | "2024年3月15日"、"近日" |
| location | 事件发生地点列表 | ["横须贺", "关岛", "南海"] |
| subject | 事件参与者列表 | ["拜登", "北约", "F-35战机", "第七舰队"] |
| action | 核心行为描述 | "进行联合军事演习" |
| original_text | 原文句子 | 完整原文描述 |

#### 2.2.3 主体分类

- **人物**：具体人名（如：拜登、马克龙）
- **组织机构**：政府、国际组织（如：北约、联合国）
- **武器装备**：军事装备（如：萨德反导系统、F-35战机）
- **部队单位**：军事编制（如：第七舰队、海军陆战队）

#### 2.2.4 功能列表

## 4. 核心算法

### 4.1 层次化事件抽取算法

#### 4.1.1 设计思想

采用两层架构，先用规则快速筛选，再用LLM精确抽取，大幅降低LLM调用成本。

```
输入文本
    │
    ▼
┌─────────────────────────────────┐
│  第一层：规则筛选（快速、零成本）  │
│  ├─ 时间表达式正则匹配            │
│  ├─ 行为动词列表检测             │
│  ├─ 主体（人物/组织/武器/部队）   │
│  └─ 判断：有主体 OR 有时间        │
└─────────────────────────────────┘
    │
    ├─ 无事件特征 → 跳过
    │
    ▼ 有事件特征
┌─────────────────────────────────┐
│  第二层：LLM精确抽取（慢速、高精度）│
│  ├─ 调用Qwen3.5-122B模型         │
│  └─ 输出结构化事件JSON            │
└─────────────────────────────────┘
    │
    ▼
合并去重 → 输出结果
```

#### 4.1.2 成本节省

| 文本类型 | 节省比例 |
|----------|----------|
| 新闻报道 | ~20% |
| 评论文章 | ~70% |
| 混合文档 | ~50% |

### 4.2 提示词设计

```
你是一个事件抽取专家。请从给定文本中抽取所有原子化事件。

每个事件包含以下要素：
- time: 事件发生的具体时间
- location: 事件发生的地点列表
- subject: 事件的参与者列表（人物、组织、武器、部队）
- action: 主体执行的核心行为
- original_text: 原文句子

要求：
1. 抽取所有事件，不要遗漏
2. 同一事件只抽取一次
3. 未提及的字段直接省略
4. subject必须列出所有参与者
5. location必须列出所有地点
6. original_text是必填字段
```

| 字段 | 类型 | 说明 |

| total_events | INT | 事件数量 |

## 6. 接口设计

### 6.1 算法服务接口

#### POST /extract

**请求体**：
```json
{
  "text": "待抽取的文本内容",
  "origin_text_id": 123
}
```

**响应体**：
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "extraction_time": "2026-04-09 15:00:00",
    "text_length": 1000,
    "paragraph_count": 5,
    "llm_calls": 3,
    "llm_calls_saved": 2,
    "total_events": 5,
    "events": [...]
  }
}
```

#### GET /health

健康检查接口。

### 6.2 后端接口

#### POST /extraction/extract

执行事件抽取。

**参数**：
- `originTextId`: 文本ID
- `force`: 是否强制重新抽取

#### GET /extraction/result/{id}

获取抽取结果。

