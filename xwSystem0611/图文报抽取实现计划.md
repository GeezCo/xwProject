## 二、图文报抽取功能实现

### 核心设计原则

**文字报 vs 图文报**:

| 维度 | 文字报 | 图文报 |
|------|--------|--------|
| 输入 | `content` 文本 | `images` URL 数组（**忽略 content**） |
| 处理方式 | 段落分割 → 多轮 LLM | **逐图调用** LLM"看图说话" |
| Event 结构 | 完整（time/location/subject/action/original_text/labels） | **简化**（subject + original_text + labels） |
| LLM 消息格式 | `{"role":"user", "content": "文本"}` | `{"role":"user", "content": [{"type":"text"}, {"type":"image_url"}]}` |
| original_text | 事件原文句子 | **对图片的文字描述** |

```python
def extract_all(self, text: str, image_urls: List[str] = None) -> Dict:
    """
    统一抽取入口
    - 文字报：image_urls 为空，走原有段落抽取流程
    - 图文报：image_urls 非空，走图像理解流程（忽略 text）
    """
    if image_urls:
        return self._extract_from_images(image_urls)
    else:
        # 原有逻辑保持不变
        return super().extract_all(text)
```

#### 1.2 新增 `_extract_from_images()` 主流程

```python
def _extract_from_images(self, image_urls: List[str]) -> Dict:
    """
    图文报抽取：逐图调用 LLM"看图说话"
    
    Args:
        image_urls: 图片 URL 列表
    
    Returns:
        {
            "events": [
                {
                    "event_id": 1,
                    "subject": ["F-35战斗机", "里根号航母"],
                    "original_text": "图中显示F-35战斗机从里根号航母甲板起飞...",
                    "labels": ["飞机", "船只"]
                }
            ],
            "total_events": 2,
            "llm_calls": 2  # 逐图调用
        }
    """
    all_events = []
    for i, img_url in enumerate(image_urls, 1):
        event = self._extract_single_image(img_url, event_id=i)
        if event:
            all_events.append(event)
    
    return {
        "extraction_time": datetime.now().strftime('%Y-%m-%d %H:%M:%S'),
        "text_length": 0,  # 图文报无文本
        "paragraph_count": len(image_urls),
        "llm_calls": len(image_urls),
        "total_events": len(all_events),
        "events": all_events,
        "labels": self._aggregate_labels(all_events),
        "entities": self._aggregate_entities(all_events)
    }
```

#### 1.3 新增 `_extract_single_image()` 单图抽取

```python
def _extract_single_image(self, img_url: str, event_id: int) -> Dict:
    """
    单图抽取：调用多模态 LLM 理解图片
    """
    prompt = self._build_image_prompt()
    
    # 构造多模态 messages
    messages = [{
        "role": "user",
        "content": [
            {"type": "text", "text": prompt},
            {"type": "image_url", "image_url": {"url": img_url}}
        ]
    }]
    
    try:
        response = self.client.chat.completions.create(
            model=self.config['llm']['model'],
            messages=messages,
            temperature=0.1,
            max_tokens=1000
        )
        
        # 解析 JSON 输出
        content = response.choices[0].message.content
        result = self._parse_image_response(content)
        result['event_id'] = event_id
        return result
        
    except Exception as e:
        logger.error(f"图片抽取失败 ({img_url}): {e}")
        return None
```

#### 1.4 新增 `_build_image_prompt()` 构造 Prompt

```python
def _build_image_prompt(self) -> str:
    """构造图像理解 prompt"""
    # 读取 subject_categories 作为识别提示
    categories_hint = "、".join(self.config['subject_categories'].keys())
    
    return f"""请识别图片中的军事相关内容，重点关注：{categories_hint}。

输出JSON格式（不要markdown代码块）：
{{
  "subject": ["识别到的主体实体"],
  "original_text": "用1-2句话描述图片内容",
  "labels": ["主体对应的分类标签"]
}}

注意：
- subject 列表中的每个实体应标注其类型（如"F-35战斗机"而非"战斗机"）
- original_text 是对整张图片的文字描述，作为事件原文
- labels 从以下分类中选择：{categories_hint}
- 如果图片中没有军事相关内容，返回空的 subject 和 labels 数组"""
```

#### 1.5 新增 `_parse_image_response()` 解析响应

```python
def _parse_image_response(self, response: str) -> Dict:
    """
    解析图像抽取的 JSON 响应
    
    返回结构：
    {
        "subject": [...],
        "original_text": "...",
        "labels": [...]
    }
    """
    import json
    import re
    
    # 方法1: 尝试直接解析 JSON
    try:
        # 提取 JSON 对象
        json_match = re.search(r'\{[\s\S]*\}', response)
        if json_match:
            result = json.loads(json_match.group())
            if 'subject' in result and 'original_text' in result:
                return result
    except json.JSONDecodeError:
        pass
    
    # 方法2: 尝试解析 ```json``` 代码块
    try:
        code_match = re.search(r'```json\s*([\s\S]*?)\s*```', response)
        if code_match:
            result = json.loads(code_match.group(1))
            return result
    except:
        pass
    
    # 兜底：返回空结构
    logger.warning(f"图像响应解析失败，返回空结构: {response[:200]}")
    return {
        "subject": [],
        "original_text": "",
        "labels": []
    }
```

#### 1.6 新增辅助聚合方法

```python
def _aggregate_labels(self, events: List[Dict]) -> List[str]:
    """从 events 中聚合所有 labels"""
    all_labels = set()
    for event in events:
        labels = event.get('labels', [])
        if labels:
            all_labels.update(labels)
    return list(all_labels)

def _aggregate_entities(self, events: List[Dict]) -> Dict:
    """从 events 中聚合 entities（按 labels 分组）"""
    from collections import defaultdict
    entities = defaultdict(set)
    
    for event in events:
        subjects = event.get('subject', [])
        labels = event.get('labels', [])
        
        # 根据 labels 分组主体
        for label in labels:
            entities[label].update(subjects)
    
    return {k: list(v) for k, v in entities.items()}
```

### 实现步骤 4: 后端层 - 创建 MinioStorageService

**文件**: `后端/src/main/java/com/qy/dch/service/MinioStorageService.java`（新建）

```java
package com.qy.dch.service;

import io.minio.MinioClient;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;

/**
 * MinIO 存储服务
 */
@Slf4j
@Service
public class MinioStorageService {

    @Autowired
    private MinioClient minioClient;

    @Value("${minio.bucket}")
    private String bucket;

    @Value("${minio.endpoint}")
    private String endpoint;

    /**
     * 构建 MinIO 对象的公网访问 URL
     *
     * @param objectName 对象名（如 "9741_1.jpg"）
     * @return 完整 URL（如 "http://36.103.234.242:9205/xianwei-images/9741_1.jpg"）
     */
    public String buildPublicUrl(String objectName) {
        if (objectName == null || objectName.isEmpty()) {
            return "";
        }
        // 构造公网 URL
        return String.format("%s/%s/%s", endpoint, bucket, objectName);
    }
}
```

## 五、注意事项

1. **模型配置**: 确认 `config.json` 使用的模型支持多模态（当前 Qwen3.5-122B-A10B 支持）
2. **图片 URL**: MinIO URL 必须是公网可访问的完整 URL
3. **错误处理**: 单张图片抽取失败不应影响其他图片
4. **性能**: 图文报抽取时间 = 图片数量 × 单次 LLM 调用时间（约 3-8 秒/图）
5. **向后兼容**: 文字报抽取逻辑保持不变
6. **日志记录**: 所有关键步骤都应有日志输出
