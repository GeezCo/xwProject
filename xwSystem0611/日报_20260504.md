# 献微系统开发日报 — 2026年5月4日

## 一、今日完成工作

### 1. 数据库与数据层

- **176服务器MySQL**新增 `images` 列（`ALTER TABLE origin_text ADD COLUMN images TEXT`）
- 创建新分类体系：一级分类「航拍报」(id=77) → 二级分类「民用报」(id=78)
- 导入 **62条图文报**数据到176服务器MySQL，包含图片文件名（JSON数组格式）
- 修正 `images` 字段格式：从纯字符串 `"xxx.png"` 修正为JSON数组 `["xxx.png"]`
- 同步242服务器MySQL数据（创建分类 + 导入图文报 + 修正格式）

### 2. 后端代码修复（3处Bug）

- **UygurController.getTextList()**：响应Map补充 `modalType`、`images`、`isExtracted` 三个字段（原来只返回sid/title/content/times/type/labels）
- **UygurController.getDetail()**：同上，补充三个缺失字段
- **Java版本冲突修复**：target目录残留JDK17编译产物（class file v61），运行时为Java 8（class file v52）导致 `UnsupportedClassVersionError`。通过 `mvnw clean compile` 强制用Java 8重新编译解决

### 3. 新增配置接口（MinIO地址可迁移化）

- **后端**：`GET /uygur/config` — 从 `application.yml` 读取 `minio.endpoint` + `minio.bucket` 拼接返回 `minioPrefix`
- **前端 store**：`home.js` 新增 `minioPrefix` 状态和 `fetchConfig()` 方法
- **前端 Home.vue**：`onMounted` 中调用 `store.fetchConfig()` 启动时获取配置
- **前端 searchResult.vue**：`MINIO_PREFIX` 从硬编码改为 `computed(() => store.minioPrefix)`
- **效果**：MinIO迁移时只需改 `application.yml`，无需重新构建前端

### 4. 接口文档更新

- `后端/接口文档.md` 第1.2节（getTextList）响应示例补充 `modalType`、`isExtracted`、`images`、`labelsJson` 字段
- 第1.3节（detail）响应示例补充 `modalType`、`isExtracted`、`images` 字段
- 新增字段说明：`modalType` 取值范围、`images` 的JSON数组格式与MinIO URL拼接规则

## 三、发现并解决的问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| getTextList/detail 响应缺少字段 | Controller手动构建Map时遗漏了新增字段 | 补充 modalType/images/isExtracted |
| typeId=78查询返回0条 | 后端连接176 MySQL，数据只导入了242 MySQL | 在176 MySQL上重新执行DDL和数据导入 |
| modalType筛选返回400 | Windows curl中文编码问题 | 用Python urllib发送正确UTF-8请求验证OK |
| Java版本冲突启动失败 | IDE用JDK17编译残留在target，运行时为Java8 | `mvnw clean compile` 强制用Java8重编译 |
| MinIO地址硬编码不利迁移 | 前端写死IP:端口/bucket | 新增 /uygur/config 接口，前端启动时动态获取 |
