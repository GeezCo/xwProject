# 离线910B ARM服务器前端部署说明

## 目标环境

- 服务器架构：ARM64 (aarch64)
- 系统：离线环境，无法访问外网
- 部署方式：Docker容器 + nginx

---

## 准备工作

### 1. 本地构建前端

在有网络的开发机器上执行：

```bash
cd 前端/qyportrait
npm install
npm run build
```

构建完成后，`dist/` 目录包含所有前端文件。

### 2. 准备nginx镜像文件

在有网络的机器上导出ARM64架构的nginx镜像：

```bash
# 拉取ARM64 nginx镜像
docker pull docker.1ms.run/library/nginx:alpine

# 导出镜像为tar文件
docker save docker.1ms.run/library/nginx:alpine -o nginx-alpine-arm64.tar

# 或者使用更小的镜像名
docker tag docker.1ms.run/library/nginx:alpine nginx:alpine-arm64
docker save nginx:alpine-arm64 -o nginx-alpine-arm64.tar
```

### 3. 准备nginx配置文件

创建 `nginx.conf` 文件：

```nginx
server {
    listen 80;
    server_name localhost;
    
    root /usr/share/nginx/html;
    index index.html;
    
    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;
}
```

## 部署步骤

### 步骤1：上传文件到服务器

使用scp或其他方式上传文件：

```bash
# 上传前端文件
scp -r dist/* user@server_ip:~/xianwei/frontend/

# 上传镜像文件
scp nginx-alpine-arm64.tar user@server_ip:~/xianwei/

# 上传nginx配置
scp nginx.conf user@server_ip:~/xianwei/
```

### 步骤2：在服务器上创建目录结构

```bash
mkdir -p ~/xianwei/frontend
mkdir -p ~/xianwei/config
```

目录结构：
```
~/xianwei/
├── frontend/          # 前端文件
│   ├── index.html
│   └── static/
│       ├── css/
│       ├── js/
│       ├── img/
│       └── fonts/
├── nginx.conf         # nginx配置
└── nginx-alpine-arm64.tar  # 镜像文件
```

### 步骤3：加载Docker镜像

```bash
# 加载镜像
docker load -i ~/xianwei/nginx-alpine-arm64.tar

# 查看镜像是否加载成功
docker images | grep nginx
```

### 步骤4：启动nginx容器

```bash
docker run -d \
    --name xianwei-frontend \
    -p 8518:80 \
    -v ~/xianwei/frontend:/usr/share/nginx/html:ro \
    -v ~/xianwei/nginx.conf:/etc/nginx/conf.d/default.conf:ro \
    nginx:alpine-arm64
```

**参数说明**：
- `-d`：后台运行
- `--name xianwei-frontend`：容器名称
- `-p 8518:80`：端口映射，外部8518映射到容器80
- `-v`：挂载卷，前端文件和配置文件

### 步骤5：验证部署

```bash
# 查看容器状态
docker ps | grep xianwei

# 测试访问
curl http://localhost:8518

# 查看容器日志
docker logs xianwei-frontend
```

## 常用管理命令

### 查看容器状态
```bash
docker ps | grep xianwei-frontend
```

### 查看容器日志
```bash
docker logs xianwei-frontend
docker logs -f xianwei-frontend  # 实时查看
```

### 重启容器
```bash
docker restart xianwei-frontend
```

### 停止容器
```bash
docker stop xianwei-frontend
```

### 删除容器
```bash
docker rm -f xianwei-frontend
```

### 更新前端文件
```bash
# 上传新的前端文件后，重启容器即可
scp -r dist/* user@server_ip:~/xianwei/frontend/
docker restart xianwei-frontend
```

### 更新nginx配置
```bash
# 上传新配置后，重启容器
scp nginx.conf user@server_ip:~/xianwei/
docker restart xianwei-frontend
```

## 故障排查

### 1. 容器无法启动

检查镜像是否正确加载：
```bash
docker images | grep nginx
```

检查配置文件是否存在：
```bash
ls -la ~/xianwei/nginx.conf
```

### 2. 页面无法访问

检查容器是否运行：
```bash
docker ps
```

检查端口是否正确映射：
```bash
docker port xianwei-frontend
```

检查防火墙：
```bash
# 查看端口是否开放
netstat -tlnp | grep 8518
```

### 3. 页面显示空白或404

检查前端文件是否完整：
```bash
ls -la ~/xianwei/frontend/
ls -la ~/xianwei/frontend/static/
```

检查index.html是否存在：
```bash
cat ~/xianwei/frontend/index.html
```

## 注意事项

1. **架构兼容性**：确保nginx镜像为ARM64架构，x86镜像无法在ARM服务器上运行

2. **镜像导出**：在有网络的ARM机器上导出镜像，或在支持多架构的机器上指定架构：
   ```bash
   docker pull --platform linux/arm64 nginx:alpine
   ```

3. **端口冲突**：确保8518端口未被占用，可根据实际情况修改端口

4. **文件权限**：确保前端文件和配置文件有正确的读取权限

5. **配置更新**：修改nginx配置后需要重启容器才能生效
