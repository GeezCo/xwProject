# 七、每日要情接口 (EventAnalysisController)

> 新增于 v1.4 (2026-06-08)

## 7.1 手动触发分析任务

```
POST http://localhost:8081/api/eventAnalysis/trigger
Content-Type: application/json
```

**请求体**:
```json
{
  "startDate": "2026-06-07",
  "endDate": "2026-06-08"
}
```

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": "分析任务已启动，请稍后查看结果"
}
```

**说明**:
- 异步执行，立即返回
- 后台逐条调用算法服务 `/eventSplit` 接口
- 已分析过的报文自动跳过（基于联合唯一索引）
- 分析结果存入 `event_analysis` 表

---

## 7.2 查询事件（按关键词筛选）

```
POST http://localhost:8081/api/eventAnalysis/query
Content-Type: application/json
```

**请求体**:
```json
{
  "startDate": "2026-06-01",
  "endDate": "2026-06-08",
  "keywords": ["美国", "俄罗斯", "北约"]
}
```

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": [
    {
      "id": 1,
      "originTextId": 9741,
      "eventTime": "2024年3月15日",
      "eventLocation": "美国白宫",
      "eventContent": "美国总统拜登签署关于人工智能的行政命令",
      "eventAnalysis": "该事件发生于2024年3月15日...",
      "analysisDate": "2026-06-07",
      "createTime": "2026-06-08 02:15:30",
      "sourceTitle": "美国AI政策动态"
    }
  ]
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 事件分析记录ID |
| originTextId | int | 原始报文ID（关联 origin_text.sid） |
| eventTime | string | 事件时间（LLM提取） |
| eventLocation | string | 事件地点 |
| eventContent | string | 事件内容（一句话摘要） |
| eventAnalysis | string | 事件深度分析（6维分析文本） |
| analysisDate | string | 分析日期（yyyy-MM-dd） |
| createTime | string | 记录创建时间 |
| sourceTitle | string | 来源报文标题（JOIN查询） |

**关键词匹配规则**:
- OR 关系：命中任意一个关键词即返回
- 匹配字段：event_content、event_analysis、报文标题
- keywords 为空数组时返回所有事件

## 7.3 查询分析状态

```
GET http://localhost:8081/api/eventAnalysis/status?date=2026-06-07
```

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "date": "2026-06-07",
    "eventCount": 45,
    "analyzed": true
  }
}
```

---

## 7.4 定时任务

**执行时间**: 每日凌晨 2:00

**执行逻辑**:
1. 查询前一天新增报文（按 `times` 字段筛选）
2. 逐条调用算法服务 `POST /eventSplit`
3. 解析事件列表，保存到 `event_analysis` 表
4. 已分析过的报文自动跳过

**Cron 表达式**: `0 0 2 * * ?`

---

## 7.5 数据表结构

```sql
CREATE TABLE event_analysis (
    id INT AUTO_INCREMENT PRIMARY KEY,
    origin_text_id INT NOT NULL,
    event_time VARCHAR(100),
    event_location VARCHAR(200),
    event_content TEXT,
    event_analysis TEXT,
    analysis_date DATE NOT NULL,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_origin_text_id (origin_text_id),
    INDEX idx_analysis_date (analysis_date),
    UNIQUE KEY uk_origin_event (origin_text_id, event_content(100))
);
```
