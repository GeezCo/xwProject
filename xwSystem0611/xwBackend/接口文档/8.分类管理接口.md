# 分类管理接口

## 8.1 接口概览

分类系统采用**多级树形结构**，支持最多 5 层层级，使用 **邻接表 + 物化路径** 混合存储方式。

### 核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 分类ID（主键） |
| name | VARCHAR(100) | 分类名称（全局唯一） |
| parent_id | BIGINT | 父节点ID（NULL=根节点） |
| level | TINYINT | 层级（1-5） |
| full_path | VARCHAR(500) | 完整路径（如 `根分类/美俄/美国/国防部`） |
| is_leaf | TINYINT(1) | 是否叶子节点（1=是，可挂载报文） |
| sort_order | INT | 同级排序 |
| description | VARCHAR(255) | 描述 |

### 接口列表

| 接口路径 | 方法 | 说明 |
|---------|------|------|
| `/api/category/tree` | GET | 获取完整分类树 |
| `/api/category/leafs` | GET | 获取所有叶子节点 |
| `/api/category/create` | POST | 新增分类节点 |
| `/api/category/update` | PUT | 更新分类（重命名、修改描述） |
| `/api/category/move` | POST | 移动分类节点 |
| `/api/category/delete/{id}` | DELETE | 删除分类（级联删除子节点） |
| `/api/category/detail/{id}` | GET | 获取分类详情 |

## 8.2 获取分类树

```
GET /api/category/tree
```

**响应**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "id": 1,
      "name": "根分类",
      "parentId": null,
      "level": 1,
      "fullPath": "根分类",
      "sortOrder": 0,
      "isLeaf": 0,
      "description": "系统根节点",
      "reportCount": 0,
      "children": [
        {
          "id": 2,
          "name": "未分类",
          "parentId": 1,
          "level": 2,
          "fullPath": "根分类/未分类",
          "isLeaf": 1,
          "reportCount": 40292,
          "children": []
        }
      ]
    }
  ]
}
```

**说明**:
- 一次性返回完整的树形结构（递归 children）
- `reportCount` 字段：该叶子节点关联的报文数量
- 根节点的 `parentId` 为 `null`

## 8.3 获取所有叶子节点

```
GET /api/category/leafs
```

**响应**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {"id": 2, "name": "未分类", "fullPath": "根分类/未分类", "level": 2},
    {"id": 5, "name": "国防部", "fullPath": "根分类/美俄/美国/国防部", "level": 4}
  ]
}
```

**用途**: 报文归类下拉框（只显示可挂载报文的叶子节点）

## 8.4 新增分类

```
POST /api/category/create
Content-Type: application/json
```

**请求参数**:
```json
{
  "name": "美国",                  // 必填，分类名称（全局唯一）
  "parentId": 3,                  // 可选，父节点ID（null表示创建根节点）
  "description": "美国相关分类"     // 可选，描述
}
```

**响应**:
```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 10,
    "name": "美国",
    "parentId": 3,
    "level": 3,
    "fullPath": "根分类/美俄/美国",
    "isLeaf": 1,
    "sortOrder": 1000
  }
}
```

**约束**:
- `name` 不能与现有分类重名
- 父节点的 `level` 必须 < 5（否则超过最大层级）
- 新增节点默认 `isLeaf=1`（叶子节点）
- 自动计算 `level` 和 `fullPath`

---

## 8.5 更新分类（重命名、修改描述）

```
PUT /api/category/update
Content-Type: application/json
```

**请求参数**:
```json
{
  "categoryId": 10,             // 必填，分类ID
  "newName": "USA",            // 可选，新名称（不传则不修改）
  "newDescription": "新描述"    // 可选，新描述（不传则不修改）
}
```

**响应**:
```json
{
  "code": 200,
  "message": "更新成功",
  "data": {
    "id": 10,
    "name": "USA",
    "fullPath": "根分类/美俄/USA",
    ...
  }
}
```

**说明**:
- 重命名时会自动更新所有子孙节点的 `fullPath`
- 新名称必须全局唯一

---

## 8.6 移动分类

```
POST /api/category/move
Content-Type: application/json
```

**请求参数**:
```json
{
  "categoryId": 10,        // 必填，要移动的分类ID
  "newParentId": 5         // 必填，新的父节点ID（null=移动到根级）
}
```

**响应**:
```json
{
  "code": 200,
  "message": "移动成功",
  "data": {
    "id": 10,
    "parentId": 5,
    "level": 3,
    "fullPath": "根分类/印度/美国",
    ...
  }
}
```

**约束**:
- 不能移动到自己的子孙节点下（防止循环引用）
- 移动后的层级深度不能超过 5
- 自动更新所有子孙节点的 `fullPath` 和 `level`

---

## 8.7 删除分类

```
DELETE /api/category/delete/{categoryId}
```

**响应**:
```json
{
  "code": 200,
  "message": "删除成功"
}
```

**说明**:
- **级联删除**：删除父节点会同时删除所有子孙节点
- 关联报文不会被删除，但 `type` 字段会变成无效ID
- 建议删除前先将报文移到其他分类
- 默认根节点（id=1, 2）不能删除

## 8.9 自动分类（导入时使用）

JSONL 导入接口（`POST /uygur/importFromJsonl`）会自动根据 `sendUnitName` 字段创建分类节点：

```jsonl
{"title": "...", "sendUnitName": "国防部"}
```

**自动分类逻辑**:
```
1. 查询是否存在 name="国防部" 且 is_leaf=1 的分类节点
2. 存在 → 直接使用
3. 不存在 → 自动创建（默认挂在"根分类"下，level=2，is_leaf=1）
```

**示例**:

导入前：
```
根分类/
└── 未分类/
```

导入后（JSONL 中有 sendUnitName="国防部"、"情报中心"）：
```
根分类/
├── 未分类/
├── 国防部/        ← 自动创建
└── 情报中心/      ← 自动创建
```

管理员后续可以通过 `/api/category/move` 接口手动调整层级关系。

## 8.10 错误响应格式

所有接口的错误响应统一格式：

```json
{
  "code": 500,
  "message": "分类名称已存在: 美国"
}
```

**常见错误**:
| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误（如分类名称为空） |
| 404 | 分类节点不存在 |
| 500 | 服务器错误（如名称重复、超过最大层级） |
