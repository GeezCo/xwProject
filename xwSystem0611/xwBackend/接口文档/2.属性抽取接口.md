# 二、属性抽取接口 (ExtractionController)

## 2.1 执行属性抽取

```
POST /extraction/extract?originTextId=9741&force=false
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| originTextId | int | 是 | - | 原始文本ID |
| force | boolean | 否 | false | 是否强制重新抽取 |

**处理流程**: 后端从数据库取文本 → 调用算法服务 `/extract` → 保存结果到 `extraction_result` 表 → 更新 `origin_text.is_extracted=1`

**响应**:

**说明**:
- 每个事件直接包含分类字段（船只、飞机、导弹等11类），值为实体名称列表

- 顶层 `labels` 为所有事件标签的汇总去重，`entities` 按分类聚合所有实体
- 分类来自 `subject_categories` 配置：船只、飞机、无人机、导弹、雷达、火炮、坦克/装甲车辆、军事基地、部队番号、人物、组织

**耗时**: 2-5分钟（取决于文本长度和LLM响应速度）

## 2.2 查询抽取结果

```
GET /extraction/result/{originTextId}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| originTextId | int | 是 | 原始文本ID（路径参数） |

**响应**: 同2.1的响应格式。未抽取时返回空数据。

## 2.3 启动批量抽取任务 ✨ 新增

```
POST /extraction/batch/start
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| startDate | string | 是 | - | 开始日期（格式：yyyy-MM-dd） |
| endDate | string | 是 | - | 结束日期（格式：yyyy-MM-dd） |
| scope | string | 否 | unextracted | 抽取范围：`unextracted`=仅未抽取，`all`=全部重新抽取 |

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "taskId": "batch_1716096000000",
    "totalCount": 150
  }
}
```

**说明**:
- 异步执行，立即返回任务ID
- 使用 `ExecutorService` 线程池处理，支持并发2个任务
- 任务状态存储在 `ConcurrentHashMap` 中

## 2.4 查询批量抽取进度 ✨ 新增

```
GET /extraction/batch/progress/{taskId}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| taskId | string | 是 | 任务ID（路径参数） |

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "taskId": "batch_1716096000000",
    "total": 150,
    "done": 45,
    "failed": 2,
    "running": true,
    "currentSid": 9876,
    "logs": [
      "任务启动，共 150 篇报文待抽取",
      "开始抽取报文 9741",
      "报文 9741 抽取成功",
      "开始抽取报文 9742",
      "报文 9742 抽取失败: 算法服务超时",
      "开始抽取报文 9876"
    ],
    "eta": 1800
  }
}
```

**字段说明**:
- `total`: 总报文数
- `done`: 已完成数
- `failed`: 失败数
- `running`: 是否正在运行
- `currentSid`: 当前正在处理的报文ID
- `logs`: 执行日志（最多保留100条）
- `eta`: 预计剩余时间（秒）

**说明**:
- 前端建议每3秒轮询一次
- 任务完成后 `running` 变为 `false`

## 2.5 停止批量抽取任务 ✨ 新增

```
POST /extraction/batch/stop/{taskId}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| taskId | string | 是 | 任务ID（路径参数） |

**响应**:
```json
{
  "code": 1,
  "msg": "任务已停止",
  "data": null
}
```

**说明**:
- 设置停止标志，任务循环检测后中止
- 已在处理的报文会完成，后续报文不再处理
