# 一、报文管理接口 (UygurController)

## 1.1 获取分类列表（已废弃）

⚠️ **该接口已废弃，请使用 `/api/category/tree`**

详见：[8.分类管理接口](8.分类管理接口.md)

```
GET /uygur/category
```

**响应** (树形结构，支持一级/二级分类):
```json
{
  "code": 1,
  "msg": "success",
  "data": [
    {
      "id": 0,
      "typeName": "开源信息",
      "parentId": null,
      "children": [
        { "id": 1, "typeName": "港口", "parentId": 0 },
        { "id": 2, "typeName": "机场", "parentId": 0 },
        { "id": 3, "typeName": "舰机过航", "parentId": 0 },
        { "id": 6, "typeName": "军事演习", "parentId": 0 }
      ]
    }
  ]
}
```

**说明**:
- 一级分类: `parentId` 为 `null`
- 二级分类: `parentId` 指向父分类的 `id`
- 当前系统有1个一级分类「开源信息」(id=0)，下辖15个二级分类

## 1.2 分页查询报文列表

```
POST /uygur/getTextList
Content-Type: application/json
```

**请求体**:
```json
{
  "pageNum": 1,
  "pageSize": 10,
  "typeId": 1,
  "modalType": "文字报"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | int | 是 | 页码，从1开始 |
| pageSize | int | 是 | 每页条数 |
| typeId | int | 否 | 二级分类ID（如6=军事演习），为空查全部 |
| modalType | string | 否 | 模态类型：文字报/图文报/声像报 |

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "pageNum": 1,
    "pageSize": 10,
    "pages": 5,
    "total": 50,
    "list": [
      {
        "sid": 9741,
        "title": "报文标题",
        "content": "报文正文...",
        "times": "2024-03-15",
        "type": 1,
        "modalType": "图文报",
        "isExtracted": 1,
        "images": "[\"9741_1.jpg\",\"9741_2.jpg\"]",
        "labelsJson": "[\"船只\",\"部队番号\"]"
      }
    ]
  }
}
```

## 1.3 获取报文详情

```
GET /uygur/detail/{sid}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sid | long | 是 | 报文ID（路径参数） |

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "sid": 9741,
    "title": "报文标题",
    "content": "报文正文全文...",
    "times": "2024-03-15",
    "type": 1,
    "modalType": "图文报",
    "isExtracted": 1,
    "images": "[\"9741_1.jpg\",\"9741_2.jpg\"]"
  }
}
```

**字段说明**:
- `modalType`: 报文模态类型，值为 `文字报`/`图文报`/`声像报`
- `images`: 图片文件名JSON数组字符串，仅图文报有值。前端拼接 MinIO URL 前缀 `http://36.141.21.176:8522/xianwei-images/` 访问图片

## 1.4 批量导入文本

```
GET /uygur/savetext
```

从服务器配置的JSON文件（`filePath`）批量导入文本到数据库。仅初始化数据时使用。

## 1.5 重置抽取状态

```
POST /uygur/resetExtracted
```

将所有报文的 `is_extracted` 重置为0，同时返回统计信息。

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "updatedRows": 150,
    "totalTexts": 200,
    "extractedTexts": 0,
    "unextractedTexts": 200
  }
}
```

## 1.6 新增二级分类

```
POST /uygur/addCategory
Content-Type: application/json
```

**请求体**:
```json
{
  "categoryName": "仿真报",
  "parentCategoryName": "侦察报"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| categoryName | string | 是 | 二级分类名称 |
| parentCategoryName | string | 是 | 一级分类名称，若不存在则自动创建 |

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "categoryId": 45,
    "categoryName": "仿真报",
    "parentId": 44,
    "parentName": "侦察报",
    "isNewParent": false
  }
}
```

**说明**:
- 若一级分类不存在，会先创建一级分类，再创建二级分类
- `isNewParent`: true表示一级分类是新创建的，false表示已存在
- 返回的 `categoryId` 是新创建的二级分类的 `sid`

## 1.7 从JSONL导入报文

```
POST /uygur/importFromJsonl
Content-Type: multipart/form-data
```

**请求参数**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| file | file | 是 | - | JSONL文件（每行一个JSON对象） |
| defaultCategoryId | integer | 否 | 2 | 默认分类ID（当报文无 sendUnitName 时使用） |

**JSONL文件格式**:
```jsonl
{"时间": "2026-01-01", "标题": "标题1", "内容": "内容1", "sendUnitName": "国防部"}
{"时间": "2026-01-02", "标题": "标题2", "内容": "内容2", "sendUnitName": "情报中心"}
{"时间": "2026-01-03", "标题": "标题3", "内容": "内容3"}
```

**分类匹配规则（按优先级）**:
1. **自动分类**：JSONL 中有 `sendUnitName` 字段 → 自动查找或创建对应的分类节点
2. **默认分类**：JSONL 中无 `sendUnitName` 字段 → 使用 `defaultCategoryId` 指定的分类（默认2=未分类）

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "totalLines": 3,
    "successCount": 3,
    "failCount": 0,
    "categoryId": 2,
    "uploadedImages": 0,
    "errors": []
  }
}
```

**说明**:
- **必填字段**：`时间`、`标题`、`内容`（支持中英文字段名）
- **可选字段**：`sendUnitName`（送报单位，用于自动分类）、`briefTypeName`（报文类型）
- **时间格式**：支持 `2026-01-01` 或 `2026年01月01日`
- **自动分类**：根据 `sendUnitName` 自动创建分类树，如 `根分类/国防部`
- **默认值**：`modal_type` = "文字报"，`is_extracted` = 0
- **错误处理**：`errors` 数组包含导入失败的行号和错误信息

**调用示例**:

```python
# 示例1：不传参数（使用默认分类 ID=2）
import requests
url = "http://localhost:8081/uygur/importFromJsonl"
files = {'file': open('test.jsonl', 'rb')}
response = requests.post(url, files=files)

# 示例2：指定默认分类 ID=5
files = {'file': open('test.jsonl', 'rb')}
data = {'defaultCategoryId': 5}
response = requests.post(url, files=files, data=data)
```

```bash
# curl 示例
curl -X POST http://localhost:8081/uygur/importFromJsonl \
  -F "file=@test.jsonl" \
  -F "defaultCategoryId=2"
```
