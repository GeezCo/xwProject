# 三、报文融合接口 (FusionController)

## 3.1 创建融合报告

```
POST /api/fusion/create
Content-Type: application/json
```

**请求体**:
```json
{
  "reports": [
    {
      "id": 9741,
      "title": "报文标题1",
      "content": "报文正文内容1...",
      "times": "2024-03-15",
      "type": 1,
      "extractionResult": {
        "events": [...],
        "entities": {...},
        "labels": [...]
      }
    },
    {
      "id": 9742,
      "title": "报文标题2",
      "content": "报文正文内容2...",
      "times": "2024-03-16",
      "type": 1,
      "extractionResult": null
    }
  ],
  "fusionType": "standard",
  "customTitle": null
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reports | array | 是 | 报文数据列表，2-10篇 |
| reports[].id | long | 否 | 报文ID |
| reports[].title | string | 否 | 报文标题 |
| reports[].content | string | 是 | 报文正文 |
| reports[].times | string | 否 | 报文时间 |
| reports[].type | int | 否 | 分类类型 |
| reports[].extractionResult | object | 否 | 抽取结果，为空则自动触发抽取 |
| fusionType | string | 否 | 融合类型，默认 "standard" |
| customTitle | string | 否 | 自定义报告标题 |

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "fusionId": 1713766800000,
    "title": "多国军事演习综合分析报告",
    "summary": "本报告综合分析了2篇报文...",
    "timeline": "[{\"time\":\"2024-03-15\",\"description\":\"美军第七舰队在菲律宾海执行演习任务\"}]",
    "content": "## 一、事件概述\n\n...(Markdown格式)",
    "entities": "[{\"category\":\"人物\",\"name\":\"拜登\",\"action\":\"2024-03-15: 拜登在白宫签署行政命令\"},{\"category\":\"组织\",\"name\":\"北约\",\"action\":\"-\"}]",
    "labels": "[\"军事\",\"国际\"]",
    "sourceIds": "[9741,9742]",
    "modelUsed": "Qwen3.5-122B-A10B",
    "createTime": "2026-04-22 10:30:00",
    "updateTime": "2026-04-22 10:30:00"
  }
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| fusionId | long | 融合报告ID（时间戳） |
| title | string | 报告标题 |
| summary | string | 报告摘要 |
| timeline | string | 事件时间线（JSON数组字符串），每项含 `time`（YYYY-MM-DD）和 `description`（**事件原文**） |
| content | string | 报告正文（Markdown格式） |
| entities | string | **目标列表表格数据**（JSON数组字符串），每项含 `category`（实体类别）、`name`（实体名称）、`action`（**时间+原文拼接**） |
| labels | string | 智能标签（JSON数组字符串） |
| sourceIds | string | 源报文ID列表（JSON数组字符串） |
| modelUsed | string | 使用的大模型名称 |
| createTime | string | 创建时间 |
| updateTime | string | 更新时间 |

**重要变更**（2026-06-02）:
- `timeline[].description`：从 `action` 改为 **`original_text`**（事件完整原文）
- `entities[].action`：从 `action` 改为 **`时间: original_text`** 拼接格式
- `entities` 数据结构：从 `{"人物": [], "组织": []}` 字典改为 **`[{category, name, action}]` 表格行数组**
- **实体归一化合并**：同一实体不同称谓自动合并（如"拜登"与"美国总统拜登"归为一行，展示最长形式）

**耗时**: 5-8分钟

## 3.2 保存融合报告

```
POST /api/fusion/save
Content-Type: application/json
```

**请求体**: 与3.1响应中的 `data` 结构相同（FusionDTO）

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": { "fusionId": 1, "title": "...", "..." : "..." }
}
```

## 3.3 查询融合报告列表

```
GET /api/fusion/list?pageNum=1&pageSize=10
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| pageNum | int | 否 | 1 | 页码 |
| pageSize | int | 否 | 10 | 每页条数 |

**响应**: `data` 直接为融合报告数组，按 `create_time DESC` 排序
```json
{
  "code": 1,
  "msg": "success",
  "data": [
    {
      "fusionId": 20,
      "title": "...",
      "summary": "...",
      "timeline": "[...]",
      "content": "...",
      "entities": "[{\"category\":\"...\",\"name\":\"...\",\"action\":\"...\"}]",
      "labels": "[...]",
      "sourceIds": "[...]",
      "modelUsed": "Qwen3.5-122B-A10B",
      "createTime": "2026-04-22 10:30:00",
      "updateTime": "2026-04-22 10:30:00"
    }
  ]
}
```

> 注意：
> - 当前实现 `data` 直接是数组，**不包含** `total`/`pages` 分页元信息
> - 老融合记录的 `entities` 可能为字典格式 `{"人物":[]...}`（2026-06-02 之前数据），新记录为表格行数组格式

## 3.4 查询融合报告详情

```
GET /api/fusion/detail/{id}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | long | 是 | 融合报告ID（路径参数） |

**响应**: 单条记录，结构同3.2

## 3.7 目标名称搜索融合 ✨ 新增

```
POST /api/fusion/searchByTarget?targetName=第七舰队&maxReports=10
```

> ⚠️ 参数通过 **URL 查询参数** 传递（`@RequestParam`），**不是** JSON 请求体。

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| targetName | string | 是 | - | 目标名称（如"第七舰队"） |
| maxReports | int | 否 | 10 | 最多返回报文数 |

**响应**:
```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "reports": [
      {
        "id": 9741,
        "title": "报文标题",
        "content": "报文正文...",
        "times": "2024-03-15",
        "type": 1,
        "extractionResult": {
          "events": [...],
          "entities": {...},
          "labels": [...]
        }
      }
    ],
    "total": 5
  }
}
```

**说明**:
- 在 `extraction_result.entities_json` 字段中搜索包含目标名称的报文
- 使用 SQL `LIKE` 查询：`WHERE entities_json LIKE CONCAT('%', #{keyword}, '%')`
- 返回的报文已包含完整的抽取结果
- 前端可直接用于融合，传递 `fusionType: 'target'` 和 `targetName`

> **删除融合报告接口暂未实现**：FusionController 中无 `@DeleteMapping`。如需删除请直接操作数据库 `fusion_report` 表。
