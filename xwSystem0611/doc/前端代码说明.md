# 前端代码说明

献微系统前端采用 Vue 3.5.13 + Vite 6.0.7 + Element Plus 2.9.1 + Pinia 2.3.0 技术栈，提供报文管理、属性抽取、报文融合和数据看板等核心功能。

---

## 一、项目概览

### 技术栈
- **核心框架**: Vue 3.5.13 (Composition API)
- **构建工具**: Vite 6.0.7
- **UI 组件**: Element Plus 2.9.1
- **状态管理**: Pinia 2.3.0
- **路由**: Vue Router
- **HTTP 客户端**: Axios
- **图表**: ECharts

### 目录结构
```
前端/qyportrait/src/
├── App.vue                    # 根组件
├── main.js                    # 应用入口
├── assets/
│   └── theme.css              # 主题样式（深色/浅色模式）
├── components/
│   ├── Login.vue              # 登录页面
│   ├── Dashboard.vue          # 数据看板页面（747行）
│   ├── searchResult.vue       # 报文列表页面（1025行，核心功能）
│   └── fusionResult.vue       # 报文融合页面（541行）
├── views/
│   └── Home.vue               # 主框架页面（导航栏、路由出口）
├── router/
│   └── index.js               # 路由配置
├── stores/
│   └── home.js                # Pinia 全局状态管理
└── utils/
    ├── request.js             # Axios 封装（拦截器）
    └── auth.js                # Token 管理工具
```

## 二、入口与路由

### 2.1 应用入口 - [main.js](../前端/qyportrait/src/main.js)

**职责**：初始化 Vue 应用，挂载插件和组件

**关键配置**：
- 注册 Element Plus UI 库（全局中文语言包）
- 挂载 Pinia 状态管理

- 加载主题样式 `theme.css`

```javascript
import { createApp } from 'vue'

import ElementPlus from 'element-plus'
import zhCn from 'element-plus/dist/locale/zh-cn.mjs'
import 'element-plus/dist/index.css'
import App from './App.vue'
import router from './router'
import './assets/theme.css'

const app = createApp(App)
app.use(createPinia())
app.use(router)
app.use(ElementPlus, { locale: zhCn })
app.mount('#app')
```

### 2.2 路由配置 - [router/index.js](../前端/qyportrait/src/router/index.js)

**路由表**：

| 路径 | 组件 | 功能 | keepAlive |
|------|------|------|-----------|
| `/` | Login.vue | 登录页 | 否 |
| `/Home` | Home.vue | 主框架页（含子路由） | 否 |
| `/Home/searchResult` | searchResult.vue | 报文列表页 | **是** |
| `/Home/fusionResult` | fusionResult.vue | 报文融合页 | 否 |
| `/Home/dashboard` | Dashboard.vue | 数据看板页 | 否 |

**路由守卫逻辑**：
```javascript
router.beforeEach((to, from, next) => {
  const token = getToken()
  if (to.path !== '/' && !token) {
    next('/')  // 未登录跳转至登录页
  } else {
    next()
  }
})
```

**说明**：
- `searchResult.vue` 设置 `meta: { keepAlive: true }`，页面状态缓存（筛选条件、分页状态保留）
- 其他页面每次进入重新初始化

**职责**：多报文融合生成综合报告

**布局结构**（两栏布局）：

```
┌───────────────────┬─────────────────────────────┐
│                   │                             │
│  左侧源报文列表   │  右侧融合报告展示区         │
```

- "从列表页选择"按钮：跳转至 `searchResult.vue`，选中后返回
- "生成融合报告"按钮：触发融合接口

#### 2. 右侧融合报告展示（第67-174行）
- **顶部操作栏**：保存、导出、返回按钮
- **融合报告内容**：
  - 标题（可编辑）
  - 摘要
  - 目标列表表格（分类、名称、行动记录）
  - 时间线（时间 + 事件描述）

1. 用户在左侧选择多个源报文
2. 点击"生成融合报告"按钮
3. 调用 `/api/fusion/create` 接口（携带源报文 ID 列表）
4. 后端调用算法服务生成融合报告
5. 前端展示融合结果（支持保存和导出）

**关键 API 调用**：
- `POST /api/fusion/create` - 创建融合报告（传入 `reports[]`）
- `POST /api/fusion/save` - 保存融合报告
- `GET /api/fusion/detail/{id}` - 查询融合报告详情
- `GET /api/fusion/export/{id}` - 导出融合报告为 Markdown

## 四、状态管理

### 4.1 Pinia Store - [stores/home.js](../前端/qyportrait/src/stores/home.js)

**职责**：全局状态管理

**状态字段**：

| 字段 | 类型 | 说明 | 初始值 |
|------|------|------|--------|
| `token` | String | 用户登录凭证 | 从 localStorage 读取 |
| `theme` | String | 主题模式（'dark' / 'light'） | 从 localStorage 读取，默认 'dark' |
| `minioPrefix` | String | MinIO 对象存储地址前缀 | 空字符串（动态获取） |
| `fusionData` | Object | 融合报告数据（跨页面传递） | null |

**方法**：

- `fetchConfig()`：从后端 `/uygur/config` 接口获取 MinIO 配置，更新 `minioPrefix`
- `setToken(token)`：保存 token 至 localStorage 并更新状态
- `clearToken()`：清除 token（退出登录）
- `setTheme(newTheme)`：切换主题并保存至 localStorage

**使用示例**：
```javascript
import { useHomeStore } from '@/stores/home'

const homeStore = useHomeStore()
await homeStore.fetchConfig()  // 获取 MinIO 配置
const imageUrl = homeStore.minioPrefix + 'images/xxx.jpg'
```

## 五、工具层

### 5.1 HTTP 客户端 - [utils/request.js](../前端/qyportrait/src/utils/request.js)

**职责**：封装 Axios，统一处理请求/响应拦截

**配置**：
- `baseURL`：从 `import.meta.env.VITE_API_BASE_URL` 读取（默认 `http://localhost:8081`）
- `timeout`：20 分钟（1200000ms，适配 LLM 长时间处理）

**请求拦截器**：
- 自动添加 `Authorization` 请求头（值为 `Bearer ${token}`）

**响应拦截器**：
- 判断 `res.data.code`：
  - `code === 1`：返回 `res.data`
  - `code === 0`：弹出错误提示，抛出异常
  - `code === 401`：清除 token，跳转登录页

## 七、关键技术点

### 7.1 主题切换实现

通过 CSS 变量实现深色/浅色主题切换，主题配置位于 [assets/theme.css](../前端/qyportrait/src/assets/theme.css)：

```css
:root[theme="dark"] {
  --bg-main: #1e1e1e;
  --color-primary: #409eff;
  /* ... */
}

:root[theme="light"] {
  --bg-main: #f5f7fa;
  --color-primary: #409eff;
  /* ... */
}
```

切换逻辑：
```javascript
document.documentElement.setAttribute('theme', newTheme)
```

---

### 7.2 图文报图片加载

图文报的图片存储在 MinIO 对象存储中，前端动态拼接完整 URL：

```javascript
const imageUrl = homeStore.minioPrefix + imagePath
// 示例：http://36.103.234.242:9000/uygur-images/ + images/20240315/xxx.jpg
```

**图片网格展示**：
```vue
<div class="images-grid">
  <img v-for="(img, idx) in currentReport.images" :key="idx" :src="minioPrefix + img" />
</div>
```

---

### 7.3 分页与 keepAlive

`searchResult.vue` 页面设置 `meta: { keepAlive: true }`，离开页面后：
- 筛选条件（分类选择、关键词、时间范围）保留
- 分页状态保留（当前页码、每页条数）
- 详情面板状态保留

用户返回页面时，无需重新筛选，直接恢复原状态。

---

### 7.4 ECharts 图表渲染

Dashboard 页面使用 ECharts 渲染分类分布和模态分布图表：

```javascript
import * as echarts from 'echarts'

const categoryChart = echarts.init(categoryChartRef.value)
const option = {
  tooltip: { trigger: 'item' },
  series: [{
    type: 'pie',
    radius: '60%',
    data: categoryDistribution.value.map(item => ({
      name: item.type || '未分类',
      value: item.cnt
    }))
  }]
}
categoryChart.setOption(option)
```

## 十、注意事项

1. **Element Plus 中文化**：已在 `main.js` 中全局配置 `zhCn` 语言包，所有日期选择器自动显示中文
2. **长超时设置**：Axios 超时设置为 20 分钟，适配 LLM 长时间处理任务
3. **MinIO 动态配置**：图文报图片地址从后端 `/uygur/config` 接口动态获取，避免硬编码
4. **批量抽取轮询**：每 3 秒轮询一次任务进度，任务完成后自动停止轮询
5. **JSONL 导入格式**：每行一个 JSON 对象，必须包含 `"标题"`、`"内容"`、`"时间"` 三个字段
6. **分类删除逻辑**：删除一级分类会级联删除其下所有二级分类（后端强制约束）

## 十一、与后端协作

前端通过 Axios 调用后端 REST API，所有接口遵循统一响应格式：

```javascript
{
  code: 1,          // 1=成功，0=失败
  data: {},         // 返回数据
  msg: "success",   // 消息描述
  flag: "true"      // 操作标志
}
```

**跨域处理**：后端已配置 CORS，允许前端域名跨域访问

**认证机制**：前端请求头携带 `Authorization: Bearer ${token}`，后端验证 token 有效性
