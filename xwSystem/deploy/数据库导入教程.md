## 数据表关系

```
text_type (分类目录)
  ↓ type = text_type.id
origin_text (报文)
  ↓ origin_text_id = origin_text.sid
extraction_result (抽取结果)  ← 系统自动生成，不需要手动插入
  ↓
fusion_report (融合报告)     ← 系统自动生成，不需要手动插入
```

只需要往 `text_type` 和 `origin_text` 两张表写数据，后两张表是系统执行抽取和融合时自动填充的。

## 表结构

### origin_text 报文表

| 字段 | 类型 | 说明 |
|------|------|------|
| sid | INT (自增主键) | 报文唯一标识 |
| title | VARCHAR(255) | 报文标题 |
| content | VARCHAR(7000) | 报文正文 |
| times | VARCHAR(255) | 报文时间 |
| type | INT | 分类ID，关联 text_type.id |
| modal_type | VARCHAR(20) | 报文模态类型：文字报 / 图文报 |
| images | TEXT | 图片文件名列表（JSON数组格式，图文报用） |
| is_extracted | TINYINT | 是否已抽取：0未抽取，1已抽取 |

### text_type 分类目录表

| 字段 | 类型 | 说明 |
|------|------|------|
| sid | INT (自增主键) | 记录ID |
| id | INT | 分类ID（业务标识） |
| type_name | VARCHAR(255) | 分类名称 |
| parent_id | INT | 父分类ID，0表示一级分类 |

## 方式 1：通过 API 导入 JSONL 文件（推荐）

后端提供了 `POST /uygur/importFromJsonl` 接口，支持上传 JSONL 文件批量导入。

### 1) 准备 JSONL 文件

每行一条 JSON 记录，文件命名如 `data.jsonl`：

```jsonl
{"title": "北约多国军事演习报告", "content": "2024年3月15日，北约在波罗的海举行联合军事演习...", "times": "2024-03-15"}
{"title": "美军太平洋舰队部署动态", "content": "美国海军第七舰队近日在西太平洋地区...", "times": "2024-03-20"}
{"title": "中东地区军事态势通报", "content": "据观察，中东地区近期军事活动频繁...", "times": "2024-03-22"}
```

每行字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| title | 是 | 报文标题 |
| content | 是 | 报文正文 |
| times | 否 | 报文时间 |

### 2) 调用导入接口

```bash
curl -X POST "http://localhost:9202/uygur/importFromJsonl" \
  -F "file=@data.jsonl" \
  -F "parentCategoryName=军事动态" \
  -F "categoryName=演习报告"
```

参数说明：

| 参数 | 必填 | 说明 |
|------|------|------|
| file | 是 | JSONL 文件 |
| parentCategoryName | 是 | 一级分类名称（不存在会自动创建） |
| categoryName | 是 | 二级分类名称（不存在会自动创建） |

返回示例：

```json
{"code": 1, "data": {"totalLines": 3, "successCount": 3, "failCount": 0, "categoryId": 15}}
```

## 方式 2：直接连接 MySQL 执行 SQL

### 连接数据库

```bash
# 方法A：进入 MySQL 容器
docker exec -it xianwei-mysql mysql -u uygur_user -puygur_2024 uygur_project

# 方法B：从宿主机连接
mysql -h 36.141.21.176 -P 9204 -u uygur_user -puygur_2024 uygur_project
```

### 插入分类

```sql
-- 查看现有分类
SELECT * FROM text_type ORDER BY parent_id, id;

-- 插入一级分类
INSERT INTO text_type (id, type_name, parent_id) VALUES (20, '军事动态', 0);

-- 插入二级分类（parent_id 指向一级分类的 id）
INSERT INTO text_type (id, type_name, parent_id) VALUES (21, '演习报告', 20);
```

### 插入文字报

```sql
INSERT INTO origin_text (title, content, times, type, modal_type) VALUES
('北约多国军事演习报告', '2024年3月15日，北约在波罗的海举行联合军事演习...', '2024-03-15', 21, '文字报'),
('美军太平洋舰队部署动态', '美国海军第七舰队近日在西太平洋地区...', '2024-03-20', 21, '文字报');
```

### 插入图文报（带图片）

```sql
-- images 字段为 JSON 数组格式的图片文件名，图片需提前上传到 MinIO 的 xianwei-images bucket
INSERT INTO origin_text (title, content, times, type, modal_type, images) VALUES
('某基地卫星图分析', '卫星图分析显示该基地近期有大规模车辆调动...', '2024-04-01', 21, '图文报', '["satellite_01.jpg","satellite_02.jpg"]');
```

### 常用查询

```sql
-- 查看报文总数
SELECT COUNT(*) FROM origin_text;

-- 查看各分类报文数量
SELECT t.type_name, COUNT(o.sid) AS cnt
FROM text_type t LEFT JOIN origin_text o ON t.id = o.type
WHERE t.parent_id != 0
GROUP BY t.type_name ORDER BY cnt DESC;

-- 查看最近导入的报文
SELECT sid, title, times, type, modal_type FROM origin_text ORDER BY sid DESC LIMIT 10;

-- 查看已抽取/未抽取数量
SELECT is_extracted, COUNT(*) FROM origin_text GROUP BY is_extracted;
```

## 方式 3：Python 脚本批量导入

适合大批量数据，直接连数据库写入：

```python
import pymysql
import json

conn = pymysql.connect(
    host='36.141.21.176',
    port=9204,
    user='uygur_user',
    password='uygur_2024',
    db='uygur_project',
    charset='utf8mb4'
)
cursor = conn.cursor()

# 先确认分类ID（假设目标分类 id=21）
CATEGORY_ID = 21

# 读取 JSONL 数据文件
count = 0
with open('data.jsonl', 'r', encoding='utf-8') as f:
    for line in f:
        line = line.strip()
        if not line:
            continue
        item = json.loads(line)
        cursor.execute(
            "INSERT INTO origin_text (title, content, times, type, modal_type) VALUES (%s, %s, %s, %s, %s)",
            (item['title'], item['content'], item.get('times', ''), CATEGORY_ID, '文字报')
        )
        count += 1

conn.commit()
print(f'导入完成: {count} 条')
cursor.close()
conn.close()
```

```bash
pip install pymysql
```

## 注意事项

1. `content` 字段最大长度 7000 字符，超长内容会被截断
2. `type` 字段必须对应 `text_type` 表中已有的分类 ID，否则前端分类筛选无法显示
3. 图文报的 `images` 字段中的图片文件名需提前上传到 MinIO 的 `xianwei-images` bucket
4. 导入后在前端刷新页面即可看到新数据
5. 支持 UTF-8、GBK、GB2312 编码的 JSONL 文件（方式 1 自动识别）
