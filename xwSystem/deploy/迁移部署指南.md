# 献微系统 — 迁移部署指南

> 基于 2026-05-07 实际部署到 910B 服务器（36.141.21.176）的经验整理。

---

## deploy 目录文件清单

`deploy/` 是完整的部署包——配置、脚本、构建产物、迁移数据全部在内。  
上传这一个目录到新机器即可部署。

```
deploy/
├── .env                      # 端口与密码配置（唯一需要手动编辑的文件）
├── docker-compose.yml        # Docker 编排（4个容器）
├── nginx.conf.template       # Nginx 反向代理模板（自动替换端口变量）
├── application-prod.yml      # 后端 Spring Boot 生产配置（自动读取.env）
├── backend.service           # systemd 服务配置（可选，需 sudo）
├── setup.sh                  # ★ 一键部署脚本
├── start.sh                  # 启动所有服务
├── stop.sh                   # 停止所有服务
├── restart.sh                # 重启所有服务
├── logs.sh                   # 交互式日志查看
├── 迁移部署指南.md             # 本文档
│
│   ── 构建产物 ──
├── dist/                     # 前端构建产物（npm run build 输出）
│   └── index.html ...
├── uygur-project-0.0.1-SNAPSHOT.jar  # 后端 JAR 包（mvn package 输出）
├── 算法/                     # 算法模块源码
│   ├── 算法服务/              #   Flask 主服务
│   ├── 数据抽取-新/           #   事件抽取算法（含 config.json）
│   └── 报文融合/              #   报文融合算法
│
│   ── 迁移数据 ──
├── init.sql                  # MySQL 数据库备份（mysqldump 输出）
└── minio_images/             # MinIO 图片备份（63张，~142MB）
```

## 一、前置条件

### 新机器要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux（Ubuntu 20.04+ 推荐） |
| 架构 | x86_64 或 ARM64 均可 |
| Docker | 20.10+（含 `docker-compose`） |
| 磁盘 | ≥10GB |
| 网络 | 能访问外网（拉镜像、pip 依赖） |

> 不需要 sudo 权限，不需要预装 Java/Python/Node.js。

### 需要迁移的数据

| 数据 | 来源 | 大小 |
|------|------|------|
| MySQL 数据库 | `uygur_project` 库，4张表 | ~42MB |
| MinIO 图片 | `xianwei-images` bucket，63张图 | ~142MB |

## 二、准备 deploy 目录（在开发机上执行）

### Step 1 — 从旧系统导出数据

#### 导出 MySQL

```bash
# 旧 MySQL 在 Docker 中：
docker exec <旧mysql容器名> mysqldump -u root -p<密码> \
  --databases uygur_project \
  --single-transaction --quick --lock-tables=false \
  > deploy/init.sql

# 旧 MySQL 是主机安装的：
mysqldump -h <旧IP> -P <旧端口> -u root -p<密码> \
  --databases uygur_project \
  --single-transaction --quick --lock-tables=false \
  > deploy/init.sql

# 验证
head -5 deploy/init.sql   # 应看到 "MySQL dump" 头
```

#### 导出 MinIO 图片

```bash
# 方法一：通过 mc 客户端
mc alias set oldminio http://<旧IP>:<旧MinIO端口> <用户名> <密码>
mc mirror oldminio/xianwei-images deploy/minio_images/

# 方法二：通过 HTTP 逐个下载（bucket 已开放匿名读取时）
mkdir -p deploy/minio_images
curl -s "http://<旧IP>:<旧MinIO端口>/xianwei-images/" | \
  grep -oP '(?<=<Key>)[^<]+' | \
  while read f; do curl -o "deploy/minio_images/$f" "http://<旧IP>:<旧MinIO端口>/xianwei-images/$f"; done
```

### Step 2 — 构建前后端

```bash
# 构建前端（需要 Node.js 18+）
cd 前端/qyportrait && npm install && npm run build
cp -r dist ../../deploy/dist

# 编译后端（需要 Java 8 + Maven）
cd 后端 && ./mvnw clean package -DskipTests
cp target/uygur-project-0.0.1-SNAPSHOT.jar ../deploy/
```

### Step 3 — 复制算法模块

```bash
# 从项目根目录执行
cp -r 算法/算法服务 算法/数据抽取-新 算法/报文融合 deploy/算法/
```

### Step 4 — 编辑 `.env`

打开 `deploy/.env`，修改 `SERVER_IP` 为新机器 IP：

```bash
SERVER_IP=<新机器IP>          # ← 必须修改
FRONTEND_PORT=9201            # ← 按需修改端口
# ... 其余一般不需要改
```

### Step 5 — 验证 deploy 目录完整性

```bash
# 确认关键文件存在
ls deploy/.env
ls deploy/docker-compose.yml
ls deploy/dist/index.html
ls deploy/uygur-project-0.0.1-SNAPSHOT.jar
ls deploy/init.sql
ls deploy/算法/算法服务/app.py
```

## 三、部署到新机器

### Step 6 — 上传 deploy 目录

```bash
# SSH 连接信息（按实际修改）
SSH="ssh -p <端口> <用户名>@<新机器IP>"
SCP="scp -P <端口>"
REMOTE="<用户名>@<新机器IP>"
XIANWEI_HOME="/home/<用户名>/xianwei"

# 创建远程目录 & 上传（只需要上传 deploy/ 一个目录）
$SSH "mkdir -p $XIANWEI_HOME"
$SCP -r deploy $REMOTE:$XIANWEI_HOME/
```

### Step 7 — 在新机器上执行一键部署

```bash
# SSH 登录新机器
ssh -p <端口> <用户名>@<新机器IP>

# 运行部署脚本
cd ~/xianwei
bash deploy/setup.sh
```

`setup.sh` 会自动完成：
1. 检查 `deploy/` 中所有必备文件是否齐全
2. 将构建产物和配置分发到运行时目录
3. 检测/下载 Java 8（如系统未安装）
4. 启动 Docker 容器（MySQL → MinIO → Algorithm → Frontend）
5. 等待各服务健康检查通过
6. 用 screen 启动后端 JAR
7. 输出健康检查报告
8. 自动恢复 MinIO 图片（如 `deploy/minio_images/` 存在）

### Step 8 — 全链路验证

```bash
source ~/xianwei/.env

echo "=== 服务健康检查 ==="
echo -n "MySQL:     "; docker exec xianwei-mysql mysqladmin ping -h localhost -u root -p$MYSQL_ROOT_PASSWORD 2>&1 | grep -q alive && echo "OK" || echo "FAIL"
echo -n "MinIO:     "; curl -sf http://localhost:$MINIO_API_PORT/minio/health/live > /dev/null && echo "OK" || echo "FAIL"
echo -n "Algorithm: "; curl -sf http://localhost:$ALGORITHM_PORT/health > /dev/null && echo "OK" || echo "FAIL"
echo -n "Backend:   "; curl -sf http://localhost:$BACKEND_PORT/actuator/health > /dev/null && echo "OK" || echo "FAIL"
echo -n "Frontend:  "; curl -sf http://localhost:$FRONTEND_PORT > /dev/null && echo "OK" || echo "FAIL"

echo ""
echo "=== 接口功能测试（通过 Nginx 代理） ==="
for path in "/uygur/category" "/uygur/config" "/uygur/detail/188" "/extraction/result/188" "/api/fusion/list"; do
    echo -n "  $path  "; curl -sf http://localhost:$FRONTEND_PORT$path -o /dev/null -w "HTTP %{http_code}\n"
done
echo -n "  POST /uygur/getTextList  "
curl -sf -X POST -H "Content-Type: application/json" -d '{"pageNum":1,"pageSize":1}' \
  http://localhost:$FRONTEND_PORT/uygur/getTextList -o /dev/null -w "HTTP %{http_code}\n"
echo -n "  MinIO图片  "
curl -sf http://localhost:$MINIO_API_PORT/xianwei-images/1008_annotated.png -o /dev/null -w "HTTP %{http_code}\n"
```

预期：全部 HTTP 200。

## 六、网络架构

```
┌──── Docker Compose ─────────────────────────────────────┐
│  Nginx(:9201)  MySQL(:9204)  MinIO(:9205)  Algo(:9203)  │
└─────┬──────────────┬───────────┬──────────────┬─────────┘
      │              │           │              │
      │ host.docker  │ localhost │ localhost    │ localhost
      │ .internal    │ :9204     │ :9205        │ :9203
      ▼              ▼           ▼              ▼
┌──────────────── 宿主机网络 ──────────────────────────────┐
│              Backend JAR (:9202)  ← screen 进程          │
└─────────────────────────────────────────────────────────┘
```

- 后端运行在宿主机，所有通信经宿主机端口映射，容器间无需直连
- Nginx 通过 `host.docker.internal` 反代后端（docker-compose.yml 中已配置 `extra_hosts`）

## 七、踩坑记录

| # | 问题 | 原因 | 解决方案 |
|---|------|------|----------|
| 1 | Nginx 容器不断重启 | 旧版 alpine 镜像是 x86，ARM64 机器不兼容 | 用 `nginx:latest`（多架构） |
| 2 | 算法容器 `Read-only file system` | 挂载 `:ro` 但 app.py 要写日志 | 去掉 `:ro`，加独立 logs 卷 |
| 3 | 后端 `Could not resolve placeholder 'filePath'` | `application-prod.yml` 缺少配置项 | 对照原 `application.yml` 补全 |
| 4 | Docker 拉取镜像失败 | 镜像源某些层 404 | 用本地已有镜像或换源 |
| 5 | MinIO 图片 HTTP 403 | 新容器空的，无 bucket | `mc mirror` 迁移 + `mc anonymous set download` |
| 6 | 无 Java 且无 sudo | 系统未装 JDK | Adoptium 免安装 JDK8 |
| 7 | nohup 后端 SSH 断开就消失 | SIGHUP 杀进程 | `screen -dmS` 启动 |

## 八、镜像选择参考

| 服务 | 默认镜像 | 国内源替代 | 说明 |
|------|----------|-----------|------|
| MySQL | `mysql:8.0` | `docker.1ms.run/library/mysql:8.0` | x86/ARM64 均有 |
| MinIO | `minio/minio:latest` | `docker.1ms.run/minio/minio:latest` | 自动适配架构 |
| Python | `docker.1ms.run/library/python:3.10-slim` | — | 算法服务 |
| Nginx | `nginx:latest` | `docker.1ms.run/library/nginx:latest` | ARM64 不要用旧版 alpine |

如果新机器无法访问 Docker Hub，在 `docker-compose.yml` 中将镜像名加前缀 `docker.1ms.run/`。
